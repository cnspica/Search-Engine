            <div class="post-text" itemprop="text"> 
             <p>I have created a trigger on the Account object that is validated only if the records are being created by an <strong>non-admin account</strong>. </p> 
             <p>In order to test this functionality I would have to set the <code>createdbyid</code>field in the test class, but when trying to do this I get the error: </p> 
             <blockquote> 
              <p>Field is not writeable: Account.CreatedById</p> 
             </blockquote> 
             <p>I know that for the <code>CreatedDate</code> field, even though normally is <strong>not writable</strong> it can be changed in a test environment. </p> 
             <p>Is the same functionality available for <code>CreatedById</code> field? </p> 
             <p>If not, are there any other workarounds that would allow me to test this scenario?</p> 
             <p>Any help would be much appreciated!</p>

             <p>There are several ways to accomplish this.</p> 
             <p>See this link: <a href="https://help.salesforce.com/apex/HTViewSolution?id=000181873&amp;language=en_US" rel="nofollow">Setting system protected fields for test code coverage</a></p> 
             <p>It says:</p> 
             <blockquote> 
              <p>There are 2 ways in which you can set system protected fields for test code coverage:</p> 
              <p>a) Using Test.loadData() </p> 
              <p>b) Using JSON.deserialize()</p> 
             </blockquote> 
             <p>I would prefer the 2nd method.</p> 
             <p><strong>Edit:</strong></p> 
             <p>As already said in comments if you only want to execute the class under any given profile you can directly use <code>System.runAs()</code> method.</p>
