            <div class="post-text" itemprop="text"> 
             <p>You must create a set of 6-sided dice and play a game with an opponent. The dice are like regular dice, but you can change the numbers on the dice. For example, regular dice have sides <code>123456</code>. You could create one with sides <code>244678</code> if you want.</p> 
             <p>The game starts with the opponent selecting any one die. After this, you also must select one (also from the same set). If both of you roll the dice, the probability of you rolling a higher number should be at least 60%.</p> 
             <p>What is the largest set of such dice that can be created?</p>

           <td class="answercell"> 
            <div class="post-text" itemprop="text"> 
             <p>You can make arbitrarily large sets of dice with this property.</p> 
             <p>Start with <a href="https://en.wikipedia.org/wiki/Nontransitive_dice#Efron.27s_dice">Efron's dice</a>:</p> 
             <ul> 
              <li>A: 4, 4, 4, 4, 0, 0</li> 
              <li>B: 3, 3, 3, 3, 3, 3</li> 
              <li>C: 6, 6, 2, 2, 2, 2</li> 
              <li>D: 5, 5, 5, 1, 1, 1</li> 
             </ul> 
             <p>A beats B, B beats C, C beats D, and D beats A with probability $\frac{2}{3}&gt;60\%$.</p> 
             <p>Now add many copies of die B, each using a different value between 2 and 4. For example, one of these new dice could have 3.5 on every side. (If the numbers need to be integers, multiply everything by a large amount first.) A will still beat all of the B dice with the same probability, and any of the B dice will still beat C. So the strategy is:</p> 
             <ul> 
              <li>If the opponent chooses A, you choose D.</li> 
              <li>If the opponent chooses any of the B's, you choose A.</li> 
              <li>If the opponent chooses C, you choose any of the B's.</li> 
              <li>If the opponent chooses D, you choose C.</li> 
             </ul>
