      <h1 itemprop="name"><a href="/questions/279254/should-we-make-country-tags-visually-distinct" class="question-hyperlink">Should we make country tags visually distinct?</a></h1>

            <div class="post-text" itemprop="text"> 
             <p>Some SE sites have questions that apply only to certain countries, or where answers should at least take laws or customs in the country in question into account. Example SE sites would be <a href="http://workplace.stackexchange.com/">Workplace</a>, <a href="http://academia.stackexchange.com/">Academia</a>, <a href="http://expatriates.stackexchange.com/">Expats</a>, <a href="http://travel.stackexchange.com/">Travel</a>, <a href="http://money.stackexchange.com/">Money</a> or <a href="http://law.stackexchange.com/">Law</a>. These sites use country tags for this purpose.</p> 
             <p>Recently, it has been proposed on Meta.Workplace to <a href="http://meta.workplace.stackexchange.com/q/3631/17125">make these country tags visually distinct</a>, to draw attention to them and avoid people posting answers that simply don't apply to a question tagged with a specific country. <a href="http://meta.workplace.stackexchange.com/a/3639/17125">The currently top-voted answer</a> suggests adding an icon to country tags, like a globe. This might be doable using the wiring for <a href="http://meta.stackexchange.com/q/30187/256777">sponsored tags</a>. The <a href="http://meta.academia.stackexchange.com/q/3327/4140">same question at Meta.Academia</a> and <a href="http://meta.travel.stackexchange.com/questions/3666/should-we-make-country-tags-visually-distinct">Meta.Travel</a> also seem to be popular.</p> 
             <p>We could, for instance, add small flags to tags: <a href="http://i.stack.imgur.com/rQ7uX.png" rel="nofollow"><img src="http://i.stack.imgur.com/rQ7uX.png" alt="flags on tags"></a> However, there seem to be <a href="http://meta.workplace.stackexchange.com/a/3639/17125">political reasons against this, as per some comments</a>, plus one could imagine disputes about flags (might the PR of China have problems with the Taiwanese flag? <a href="https://en.wikipedia.org/wiki/Flag_of_Saudi_Arabia" rel="nofollow">Saudi Arabia's flag</a> with the <em>Shahada</em> may be problematic for religious reasons.). A stylized globe would be more politically neutral. I'll set up answers for voting.</p> 
             <p>I'd like to ask the good folks at SE to allow adding such an icon to country tags. It could then be up to the individual sites whether or not to actually use this feature. (I'd assume people at Travel or Expats don't overlook a country tag as often as they might at Workplace or Academia.)</p>

             <p>Regarding the comment that Arqade.SE might want game tags to be made visually distinct and programming stack exchanges might want language tags to be visually distinct, I am not sure how often a question tagged <a href="/questions/tagged/c%2b%2b" class="post-tag" title="show questions tagged 'c++'" rel="tag">c++</a> gets a java answer on programming related sites.</p>

           <td class="answercell"> 
            <div class="post-text" itemprop="text"> 
             <p>Currently in meta, there are required tags: <a href="/questions/tagged/feature-request" class="post-tag required-tag" title="show questions tagged 'feature-request'" rel="tag">feature-request</a>, <a href="/questions/tagged/bug" class="post-tag required-tag" title="show questions tagged 'bug'" rel="tag">bug</a>, <a href="/questions/tagged/discussion" class="post-tag required-tag" title="show questions tagged 'discussion'" rel="tag">discussion</a>, <a href="/questions/tagged/support" class="post-tag required-tag" title="show questions tagged 'support'" rel="tag">support</a>. These are highlighted by a different color. E.g. contrast with <a href="/questions/tagged/tags" class="post-tag" title="show questions tagged 'tags'" rel="tag">tags</a>. This makes it easy to see the most important tag for the question. Changing the color and not providing an icon would retain sponsored tags' privileged nature while making country tags more prominent. </p> 
             <p>There are two problems with things like country tags. First, people tend to leave them off. Second, people sometimes ignore them. In regards to game (Arqade) and programming language (Code Review, etc.), the first is the larger problem. In my opinion this suggests that country tags are not the only tags that should be promoted. Thus I would suggest a system that allows for sites to specify which tags they want to differentiate from the others rather than a system that only works for country tags. </p> 
             <p>Note that I am not arguing that these tags should be required as with the tags on the meta sites, just strongly encouraged. It seems to me that it is possible to ask neutral questions on most sites (Code Review might be an exception). </p>
