            <div class="post-text" itemprop="text"> 
             <p>Using your typical duplicate answer as a reference:</p> 
             <p><img src="http://i.stack.imgur.com/BVDyQ.png" alt="Typical duplicate."></p> 
             <p>I think that the "has an answer" text of the duplicate section should either be linked to the duplicate answer.</p> 
             <p>Or, if that's untenable, clicking the "has an answer" text could cause the browser to scroll up to the "This question already has an answer here" box and highlight it or something like that.</p> 
             <p>Why?</p> 
             <ol> 
              <li>Banner blindness is a real thing, and so some people might miss the block up top. </li> 
              <li>It's not intuitive to tell someone there's a link but have that link be somewhere else. Even if I know that I would need to scroll up from the "marked as duplicate" section to get to the duplicate link, that's just not good UX.</li> 
              <li>No real change in front end design other than wrapping three words with an anchor. Unless this would be a huge pain to implement on the back end, it's a minimal change with a big upside.</li> 
             </ol>

           <td class="answercell"> 
            <div class="post-text" itemprop="text"> 
             <p>I think this is a problem as well. </p> 
             <p>First, no one reads the answer before they read a question. So placement before the question text is counter intuitive. It asks users to back up, rewind,and look <em>before</em> the question for an answer. </p> 
             <p>The example used in this thread/question/answer is a simple one - easily viewable "above the fold" as it were. However, imagine a longer question which a user scrolls to read.... they get to the bottom, see it's marked as a duplicate.... then have <em>no clue</em> where the duplicate may be. </p> 
             <p>While not expressed in the best of ways, we had a user post a comment on meta.graphicdesign.stackexchange.com who was <em>clearly</em> frustrated by this very thing. I can imagine for the reason I've posted above. </p> 
             <p>Second, The placement of the duplicate link is positioned, and often colored, in a manner which promotes user blindness. I've found this true across all SE sites. It's visually a banner, and as such easily falls into the "content blindness" experience many users get while viewing any site. </p> 
             <p>Combine #1 and #2 above and it's completely disregarded by many new users more than it's noticed.</p> 
             <p>Sure, those who have experience with SE sites are used to it and know it's there and overcome the "banner blindness" consciously when we need to.</p> 
             <p>But that does not help new users.</p> 
             <p>The link should be moved, in my opinion, and be made visually more prominent.</p> 
             <p><strong>After</strong> the question text, within the actual question, would be one of the optimum positions in my opinion.</p> 
             <p><strong>Possible Example...</strong></p> 
             <p><img src="http://i.stack.imgur.com/MShTU.jpg" alt="enter image description here"></p> 
             <p>This would <em>clearly</em> allow the message and link to be seen after a user has read the question, which is when they are seeking the answer.</p> 
             <p>Even moving it below the tags, and above the "marked as" message would be better - provided it is visually different than the "marked as" message below it.</p>

           <td class="answercell"> 
            <div class="post-text" itemprop="text"> 
             <p>For me, the word <code>Duplicate</code> itself should be the link itself to the question and the <code>has an answer</code> part should be linked to the accepted answer or the top answer of the original question to avoid confusion. That is just for better UX and user experience.</p> 
             <p>Like @joojaa, the bottom block should be a complete sentence:</p> 
             <blockquote> 
              <p>Marked as duplicate of {insert link here} by so and so</p> 
             </blockquote> 
             <p>This should be to avoid confusion and to make a more visual statement to other users. Now if we add the features suggested by me, the top block (just assume some other question is a duplicate of this one) would be like this:</p> 
             <blockquote> 
              <h3>This question already <a href="http://meta.stackexchange.com/questions/239097/link-to-duplicate-answer-in-marked-as-duplicate-section">has an answer</a> here: <br></h3> 
              <p><a href="http://meta.stackexchange.com/questions/239097/link-to-duplicate-answer-in-marked-as-duplicate-section">Link to duplicate answer in 'marked as duplicate' section</a> <strong><em>3 answers</em></strong></p> 
             </blockquote> 
             <p>And the bottom one:</p> 
             <blockquote> 
              <p>Marked as duplicate of <a href="http://meta.stackexchange.com/questions/239097/link-to-duplicate-answer-in-marked-as-duplicate-section">Link to duplicate answer in 'marked as duplicate' section</a> by so and so by so and so</p> 
             </blockquote> 
             <hr> 
             <p>Here is the offending code for the above text feature for the original one:</p> 
             <p> </p> 
             <pre><code>&lt;div class="question-status question-originals-of-duplicate"&gt;
    &lt;p&gt;This question already has an answer here:&lt;/p&gt;
    &lt;ul&gt;
        &lt;li&gt;
            &lt;a href="/questions/253450/new-profile-pages-are-out-bugs-and-feedback-master-list" dir="ltr"&gt;New profile pages are out - bugs and feedback master list&lt;/a&gt;
                &lt;span class="question-originals-answer-count"&gt;
                    75 answers
                &lt;/span&gt;
        &lt;/li&gt;
    &lt;/ul&gt;
&lt;/div&gt;
</code></pre> 
             <p>and the one for the bottom:</p> 
             <pre><code>&lt;/div&gt;
&lt;/td&gt;
        &lt;/tr&gt;
                    &lt;tr&gt;
            &lt;td class="special-status" colspan="2"&gt;
                &lt;div class="question-status"&gt;
        &lt;h2&gt;                    &lt;b&gt;marked&lt;/b&gt; as duplicate by &lt;a href="/users/21960/al-e"&gt;Al E.&lt;/a&gt;, &lt;a href="/users/245167/infinite-recursion"&gt;Infinite Recursion&lt;/a&gt;, &lt;a href="/users/153355/tchrist"&gt;tchrist&lt;/a&gt;, &lt;a href="/users/152515/doppelgreener"&gt;doppelgreener&lt;/a&gt;, &lt;a href="/users/140890/martijn-pieters"&gt;Martijn Pieters&lt;/a&gt; &lt;span dir="ltr"&gt;&lt;span title="2015-04-17 12:20:39Z" class="relativetime"&gt;Apr 17 '15 at 12:20&lt;/span&gt;&lt;/span&gt;
</code></pre> 
             <p></p> 
             <p>Now all we need is to change the above to add some links there and there and problem solved! (Question used is my own, which has been marked as duplicate)... And if we edit this a little bit more, we can change this:</p> 
             <pre><code>&lt;p&gt;This question already has an answer here:&lt;/p&gt;
</code></pre> 
             <p>To this: </p> 
             <pre><code>&lt;p&gt;This question already &lt;a href="&lt;insert question link"&gt;has an answer&lt;/a&gt; here:&lt;/p&gt;
</code></pre> 
             <p>And this:</p> 
             <pre><code>&lt;b&gt;marked&lt;/b&gt; as duplicate by &lt;a href="/users/21960/al-e"&gt;Al E.&lt;/a&gt;, &lt;a href="/users/245167/infinite-recursion"&gt;Infinite Recursion&lt;/a&gt;, &lt;a href="/users/153355/tchrist"&gt;tchrist&lt;/a&gt;, &lt;a href="/users/152515/doppelgreener"&gt;doppelgreener&lt;/a&gt;, &lt;a href="/users/140890/martijn-pieters"&gt;Martijn Pieters&lt;/a&gt; &lt;span dir="ltr"&gt;&lt;span title="2015-04-17 12:20:39Z" class="relativetime"&gt;Apr 17 '15 at 12:20&lt;/span&gt;&lt;/span&gt;
</code></pre> 
             <p>To this:</p> 
             <pre><code>&lt;b&gt;marked&lt;/b&gt; as duplicate of &lt;a href={insert link to question here}&gt;{insert question title here}&lt;/a&gt; by &lt;a href="/users/21960/al-e"&gt;Al E.&lt;/a&gt;, &lt;a href="/users/245167/infinite-recursion"&gt;Infinite Recursion&lt;/a&gt;, &lt;a href="/users/153355/tchrist"&gt;tchrist&lt;/a&gt;, &lt;a href="/users/152515/doppelgreener"&gt;doppelgreener&lt;/a&gt;, &lt;a href="/users/140890/martijn-pieters"&gt;Martijn Pieters&lt;/a&gt; &lt;span dir="ltr"&gt;&lt;span title="2015-04-17 12:20:39Z" class="relativetime"&gt;Apr 17 '15 at 12:20&lt;/span&gt;&lt;/span&gt;
</code></pre> 
             <p>The above are examples but it should be how things should go applying the features and making BalusC satisfied...</p>
