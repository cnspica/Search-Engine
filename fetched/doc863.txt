             <blockquote> 
              <p>Update: <a href="http://meta.stackoverflow.com/questions/308875/new-navigation-release-candidate">A Release Candidate has been announced</a></p> 
              <p>Update: We've heard you! <a href="http://meta.stackexchange.com/questions/259220/new-navigation-alpha-2-released">Alpha 2 is now released</a>. Please let us know what you think.</p> 
             </blockquote> 
             <p>As <a href="http://meta.stackexchange.com/questions/251095/let-s-improve-our-site-navigation">announced 6-8 weeks ago...</a></p> 
             <p>In a major effort to <del>move as much UX cheese as possible in a single deployment</del> significantly improve the site browsing experience, we have deployed a new version of the site navigation to Stack Overflow, Meta Stack Overflow and Meta Stack Exchange.</p> 
             <p><img src="http://i.stack.imgur.com/4m8Ptm.png" alt="The great Parmesan earthquake of 2012"></p> 
             <p>The catch is that this is only visible to users that explicitly enable it.</p> 
             <h1>What's new?</h1> 
             <p><img src="http://i.stack.imgur.com/3OlCK.png" alt="screenshot"></p> 
             <p>This is the list of major changes we're introducing.</p> 
             <h2>Unified home page</h2> 
             <p>All major question lists have been merged into a single page, the home page. This page has three tabs: <strong>new</strong>, which shows fresh content; <strong>popular</strong>, which shows the top posts over time ranges; <strong>need answer</strong> which helps you find stuff to answer.</p> 
             <p>Almost all of these lists can be filtered by tag (we are working on the remaining ones). All of them allow different sorts and sets of questions. Finally there are further options to further customize your experience.</p> 
             <h2>Simplified top navigation</h2> 
             <p><strong>Questions</strong> is now highlighted whenever we show a list of questions, including the home page. <strong>Unanswered</strong> has been removed.</p> 
             <h1>FAQ</h1> 
             <p><strong>How do I get this?</strong></p> 
             <p><img src="http://i.stack.imgur.com/EQ4SE.png" alt="the new &quot;New Navigation&quot; panel"></p> 
             <p>Go to your <a href="/users/preferences/current">user preferences page</a>, look at the bottom. Check the checkbox, or un-check it to go back to normal. Checking the box puts you in a waiting list. </p> 
             <p>We allow people in batches to verify behavior under load, ensure we've fresh eyes as we fix the inevitable bugs and so on. </p> 
             <p>There's a five minute cache expiration time, so any change you make might take a little to propagate.</p> 
             <blockquote> 
              <p><del><strong>Update</strong>: As we have 500 active testers now (you are <em>amazing</em>!), we will delaying new alpha activations until we roll out a set of fixes. Opting in now ensures you'll be in the next few hundreds to see the next version.</del></p> 
             </blockquote> 
             <hr> 
             <blockquote> 
              <p><strong>Update</strong>: We have activated 500 more testers after deploying a set of fixes. There are around 1000 testers active now. There are 3000 people in the waiting list, gosh! You might need to wait a little bit.</p> 
             </blockquote> 
             <p><strong>How can I help?</strong></p> 
             <p>Here's what we really want to know :-)</p> 
             <ul> 
              <li>that we are not breaking stuff you used to do</li> 
              <li>how easy is to find questions to answer, <em>especially if you are a new user</em></li> 
              <li>any ideas for improvement and feature requests</li> 
              <li>bug reports</li> 
             </ul> 
             <p>If there is anything you particularly <em>hate</em>, but it's not a <em>breaking behavior</em>, please think of a better alternative and suggest it as a feature request. &lt;3</p> 
             <h1>Please read this</h1> 
             <blockquote> 
              <p><strong>How do I get back to you?</strong></p> 
              <p>Use this question for general considerations only. Please use separate questions tagged with <a href="/questions/tagged/new-nav" class="post-tag" title="show questions tagged 'new-nav'" rel="tag">new-nav</a> <a href="/questions/tagged/bug" class="post-tag required-tag" title="show questions tagged 'bug'" rel="tag">bug</a> or <a href="/questions/tagged/new-nav" class="post-tag" title="show questions tagged 'new-nav'" rel="tag">new-nav</a> <a href="/questions/tagged/feature-request" class="post-tag required-tag" title="show questions tagged 'feature-request'" rel="tag">feature-request</a> as appropriate.</p> 
             </blockquote> 
             <p>It's impossible to manage large amounts of bug reports or feature requests if you answer here. We periodically go through the list and ask answerers to move their bug/feature answers to a question and remove the post, and you likely will lose rep once the answer is gone.</p> 
             <p><strong>What are the next steps?</strong></p> 
             <p><del>We will spend the next few days hot fixing bugs which emerge. New ideas and breaking behaviors will be considered for the next iteration.</del></p> 
             <hr> 
             <p>Thanks for all the feedback, we've planned the next iteration now, all the requests we're going to tackle in it are marked as <a href="/questions/tagged/status-planned" class="post-tag moderator-tag" title="show questions tagged 'status-planned'" rel="tag">status-planned</a>. The ones tagged <a href="/questions/tagged/status-deferred" class="post-tag moderator-tag" title="show questions tagged 'status-deferred'" rel="tag">status-deferred</a> will be reconsidered in the next cycle. Obviously feel free to test and contribute more, the new homepage is not going away.</p>

             <p>I think part of this was the way it used to work, and part of it not the way it used to work. For example, it used to remember if you were on "votes", "frequently asked", etc. but it never remembered tags for example. I am not sure if there is a need to have it always default to newest, but it is a lot easier to click there (and I am so used to clicking there to refresh in the old UI) that it could at least be considered for retaining the current settings (or even just issuing the same url on click).</p>

             <p><del>I don't even realize the steps I took to see this, so I can't reproduce it at the moment</del>, but there was a point in which I visited the questions page but had no tag filter dropdown. The other filters were present; I clicked the "questions" button again, and the tag filter returned. Perhaps I'm missing something obvious, but that seemed a lot like a bug to me. </p> 
             <p>I can now offer the steps where the "filter" disappears:</p> 
             <ol> 
              <li>If open a fresh browser and navigate to stackoverflow.com, the "New" questions opens up by default with no tag filter, showing "recommended" posts and ordered by "newest". </li> 
              <li>Clicking the "Questions" button stays on "New" questions, adds the filter tag (defaulting to "all"), and shows "all" posts - still ordered by "newest."</li> 
              <li>Changing the "filter" dropdown to "favorites" filters just as one would expect and appears to work just fine.</li> 
             </ol> 
             <p>So this is all behavior within the "new" questions view - some with the tag filter, some without, and when it filters, it filters correctly, so I don't think it's a matter of "you can't filter this list yet." </p> 
             <p>Perhaps I'm alone in this, but I just wouldn't expect the "filter" (or "show" or "order") dropdowns to just disappear or reappear unless I changed to a list that didn't support that functionality at all.</p>
