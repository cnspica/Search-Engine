            <div class="post-text" itemprop="text"> 
             <p>I've just recently started to implement support for <a href="http://docs.oasis-open.org/xliff/xliff-core/v2.0/xliff-core-v2.0.html" rel="nofollow">XLIFF 2.0</a> in a software product.</p> 
             <blockquote> 
              <p>XLIFF (XML Localisation Interchange File Format) is an XML-based format created to standardize the way localizable data are passed between tools during a localization process and a common format for CAT tool files. – <a href="https://en.wikipedia.org/wiki/XLIFF" rel="nofollow">Wikipedia</a></p> 
             </blockquote> 
             <p>My questions (and perhaps answers) will be of technical nature focused on implementing (programming for) XLIFF.</p> 
             <h3>Question example</h3> 
             <blockquote> 
              <p>What is the driving motivation for an <em>Extractor</em> tool to generate multiple <code>&lt;segment&gt;</code> elements in a <code>&lt;unit&gt;</code> as opposed to generate multiple <code>&lt;unit&gt;</code> elements with just a single <code>&lt;segment&gt;</code>? </p> 
              <p>Is there a general recommendation available for how an <em>Extractor</em> should generate segments?</p> 
             </blockquote> 
             <p><strong>Which StackExchange site is the best match for this?</strong></p> 
             <hr> 
             <p>This question is not a duplicate of "<em><a href="http://meta.stackexchange.com/q/129598/255624">Which computer science / programming Stack Exchange do I post in?</a></em>" because:</p> 
             <ul> 
              <li><p>I'm asking for expertise in the XLIFF 2.0 standard.</p></li> 
              <li><p>I believe that a similar question could be asked about LaTeX in <a href="http://tex.stackexchange.com">TeX</a> or perhaps in <a href="http://english.stackexchange.com">English</a>.</p></li> 
              <li><p>The example question is not based on opinion. An expert in XLIFF could probably explain:</p> 
               <ul> 
                <li>Why one way is better than the other; or</li> 
                <li>Why one way is actually illegal; and/or </li> 
                <li>Why both options exist in the standard and how they are different.</li> 
               </ul></li> 
              <li><p><a href="http://programmers.stackexchange.com">Programmers</a> is not a good fit.</p></li> 
              <li><p>If there were a site for <em>Computer Aided Translation</em> or <em>Translators</em> then that would probably be a good fit. </p></li> 
             </ul>
