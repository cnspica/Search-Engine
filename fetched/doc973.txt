            <div class="post-text" itemprop="text"> 
             <p>I'm working with InstallAnywhere, an old program for creating installers. I would like to install a particular file, only in case of Windows XP and Windows Vista.</p> 
             <p>Due to InstallAnywhere limitations, this is not possible. I can only decide to delete the file once it has already been installed, based on a rule.</p> 
             <p>Within that rule, I can check for the platform on which I'm running, which gives me the possibility to check for Windows XP, Windows Vista, Windows 7, etc. but not for Windows 10. I can't say "Perform this action when the system is not XP or Vista", so I need to say "Perform this action on all those platforms, which are not XP or Vista".</p> 
             <p>However, I can launch command-line commands and catch the result, so here's my question: is there a command which I can use for determining if I'm working on a Windows 10 system?</p>

             <h3>Is there a command to determine if I'm working on a Windows 10 system?</h3> 
             <p>You can use <code>wmic</code>.</p> 
             <p>The following command will return the Windows version.</p> 
             <pre><code>wmic os get Caption | findstr /v Caption
</code></pre> 
             <p>Example output:</p> 
             <pre><code>F:\test&gt;wmic os get Caption | findstr /v Caption
Microsoft Windows 7 Home Premium
</code></pre> 
             <p>If you want a little more information, you can use the following batch file (GetOS.cmd), which will retrieve and display:</p> 
             <ul> 
              <li>Operating System Version</li> 
              <li>Service Pack Major Version</li> 
              <li>Architecture (64 or 32 bit)</li> 
             </ul> 
             <pre class="lang-bsh prettyprint-override"><code>@echo off
setlocal
setlocal enabledelayedexpansion
set _os=
set _sp=
rem use findstr to strip blank lines from wmic output
rem get OS
for /f "usebackq skip=1 tokens=3" %%i in (`wmic os get caption ^| findstr /r /v "^$"`) do (
  set "_os=%%i"
  )
rem get Service Pack
for /f "usebackq skip=1 tokens=*" %%i in (`wmic os get ServicePackMajorVersion ^| findstr /r /v "^$"`) do (
  set "_sp=%%i"
  )
rem get Architecture
for /f "usebackq skip=1 tokens=*" %%i in (`wmic OS get OSArchitecture ^| findstr /r /v "^$"`) do (
  set "_bits=%%i"
  )
echo Operating System: %_os%
echo Service Pack Major Version: %_sp%
echo Architecture: %_bits%
endlocal
</code></pre> 
             <p>The OS Version is stored in <code>%_os</code>, Service Pack Major Version is stored in <code>%_sp%</code>, and the Architecture is stored in <code>%_bits%</code>.</p> 
             <p>Notes:</p> 
             <ul> 
              <li><p>Not completely tested as I don't have all the OS and Service Pack combinations to test it with.</p></li> 
              <li><p>The <code>for</code> command retrieves only the 3rd part (token) of the OS. This will work for the desktop versions (if you want to distinguish Server 2008 from other versions you will need to find another solution).</p></li> 
              <li><p><code>%_os</code> will be set to one of the following values: <code>Server</code>, <code>Vista</code>, <code>7</code>, <code>8</code>, <code>8.1</code> or <code>10</code>.</p></li> 
             </ul> 
             <p>Example output:</p> 
             <pre><code>F:\test&gt;GetOS
Operating System: 7
Service Pack Major Version: 1
Architecture: 64-bit

F:\test&gt;
</code></pre>

              <li><a href="http://ss64.com/nt/">An A-Z Index of the Windows CMD command line</a> - An excellent reference for all things Windows command line related.</li> 
              <li><a href="http://ss64.com/nt/findstr.html">findstr</a> - Search for strings in files.</li> 
              <li><a href="http://ss64.com/nt/for_cmd.html">for /f</a> - Loop command against the results of another command.</li> 
              <li><a href="http://ss64.com/nt/if.html">if</a> - Conditionally perform a command.</li> 
              <li><a href="http://ss64.com/nt/wmic.html">wmic</a> - Windows Management Instrumentation Command.</li> 
             </ul>

           <td class="answercell"> 
            <div class="post-text" itemprop="text"> 
             <p>I don't have any experience with InstallAnywhere but there are two options to see what OS is installed using a command-line.</p> 
             <ol> 
              <li><p>Using Cmd</p> <pre><code>systeminfo.exe
</code></pre> <p>Using the command <code>systeminfo.exe</code>, you can see result of under the field <code>OS Name</code>. This may give you more information than you require.</p> <p>More information on this command can be found in <a href="https://technet.microsoft.com/en-gb/library/bb491007.aspx" rel="nofollow">Microsoft TechNet</a>.</p></li> 
              <li><p>Powershell</p> <p>Alternatively you could use the following Powershell script which will return the exact information you want. </p> <pre><code>Get-CimInstance Win32_OperatingSystem | Select-Object  Caption | ForEach{ $_.Caption }
</code></pre></li> 
             </ol>
