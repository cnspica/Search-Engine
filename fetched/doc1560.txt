            <div class="post-text" itemprop="text"> 
             <p>Simple example: <a href="http://stackoverflow.com/questions/17209770/better-way-to-import-java-script-files-and-css-files-using-cdn-or-from-local-mac">better way to import java script files and css files using CDN or from local machine</a></p>

             <p>However, SO won't allow me to flag the other as a duplicate because it's on another site. I don't want to close as off-topic; should be on programmers.se, because it might lead the asker to post his question on programmers.se or a mod to move the question over there.</p> 
             <p>Adding a comment + closing as off-topic is the best thing I can think of, but it feels "incomplete".</p> 
             <p>Thoughts?</p>

             <p>The Stack Overflow version of the question is <em>not</em> off topic for Stack Overflow, closing it just because a very similar question exists on another site wouldn't make much sense. The sites overlap a bit in expertise, some questions are perfectly on topic on both. However, there's a <a href="http://stackoverflow.com/questions/5257923/how-to-load-local-script-files-as-fallback-in-cases-where-cdn-are-blocked-unavai">very good duplicate</a> of it right here on Stack Overflow, so it won't really stay open for long.</p>

Assuming the moderators agreed with your flag, your comment would have quickly pointed us to the duplicate once the question reached our shores. </p>

           <td class="answercell"> 
            <div class="post-text" itemprop="text"> 
             <p>I feel that the current solution of "just put a link in the comments" is wrong for two reasons.</p> 
             <p>Firstly anyone digging through to answer old questions will stumble upon it as an unanswered question and may start working on a solution. Yes I know we should all read the comments but the fact of the matter when you look up the question it states "0 answers" which is false.</p> 
             <p>Secondly, and this is the more important case, when users are seeking answers and perform a search they too will see "0 answers". This makes the site less user friendly. Its bad enough that people tend to answer questions in the comments section, do we really need to force people to do it now?</p> 
             <p>Why wouldn't you allow from cross site answers when you already allow for duplicates. When something is flagged duplicate its not instantly deleted. Its flagged, a note is put on it stating its a duplicate an users can see see it and navigate to a solution. The current "solution" feels more like a work around for the fact that the site has a bug (sites can't communicate with each other), rather than being an actual solution. If more sites are going to be added with specific scope (see how there is vi.se and emacs.se and yet there are tons of so.se posts that already answered their questions), this really should be resolved much better. Otherwise you're going to have lots of duplicate effort or lots of questions with no answers.</p>
