            <div class="post-text" itemprop="text"> 
             <p><a href="https://github.com/rupa/z/"><code>z</code></a> is a very nice "plugin" for <code>bash</code> or <code>zsh</code> (it's included in <a href="http://ohmyz.sh/">Oh My Zsh</a>) which keeps track of directories you <code>cd</code> to, and allows you to quickly switch to directories using parts of their names, based on how frequently you use them.</p> 
             <p>So after a little while, it will know that</p> 
             <pre><code>z git-repo
</code></pre> 
             <p>should <code>cd</code> to the <code>git-repo</code> directory you use most often. You can qualify that with part of the path, <em>e.g.</em></p> 
             <pre><code>z d git-repo
</code></pre> 
             <p>or even</p> 
             <pre><code>z d git
</code></pre> 
             <p>I've found this to be more useful than aliases or even <code>CDPATH</code> because it learns on its own and adapts to my changing habits.</p>

</code></pre>

            <div class="post-text" itemprop="text"> 
             <p>Just to mention another useful tool, there is the <code>pushd</code> builtin. With</p> 
             <pre><code>$ pushd dirname
</code></pre> 
             <p>the current directory will be pushed on the directory stack (you can look at that stack with <code>dirs</code>) and the current directory will be changed to <code>dirname</code>. You can later change back to the latest (top-most in the stack) directory using the <code>popd</code> builtin command.</p> 
             <p>This should work at least in bash, zsh, csh and tcsh.</p> 
             <p>If you just want to change to the previous directory you have been in, <code>cd -</code> comes to help.</p>

             <blockquote> 
              <p>The command you are looking for is <code>pushd</code> and <code>popd</code>.</p> 
              <p>You could view a practical working example of <code>pushd</code> and <code>popd</code> from <a href="http://linux.101hacks.com/cd-command/dirs-pushd-popd/" rel="nofollow">here</a>.</p> 
              <pre><code>mkdir /tmp/dir1
mkdir /tmp/dir2
mkdir /tmp/dir3
mkdir /tmp/dir4

cd /tmp/dir1
pushd .

cd /tmp/dir2
pushd .

cd /tmp/dir3
pushd .

cd /tmp/dir4
pushd .

dirs
/tmp/dir4 /tmp/dir4 /tmp/dir3 /tmp/dir2 /tmp/dir1
</code></pre>

            <div class="post-text" itemprop="text"> 
             <p>For coding stuff I have always just used environment variables to store commonly used code directories.</p> 
             <p>In a bash file that gets executed do:</p> 
             <pre><code>export PROJECTNAMEDIR=/opt/too/long/path/to/type/every/time/git-repo
</code></pre> 
             <p>Then you can do stuff like <code>cd $PROJECTNAMEDIR</code> or <code>git checkout $PROJECTNAMEDIR</code> and also variables will autocomplete with bash.</p>
