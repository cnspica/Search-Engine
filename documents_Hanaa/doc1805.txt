      <h1 class="blog-title"><a href="http://blog.superuser.com/2016/02/07/iso-files-optical-drives-and-bootable-flash-drives/">ISO files, optical drives and bootable flash drives</a></h1> 
      <div class="meta-info">
       <span class="postedDate">February 7, 2016</span> by 
       <a href="http://blog.superuser.com/author/jiggunjer/" title="Posts by jiggunjer" rel="author">jiggunjer</a>. 
       <a href="http://blog.superuser.com/2016/02/07/iso-files-optical-drives-and-bootable-flash-drives/#comments">3 comments</a> 
      </div> 
      <div class="post-text"> 
       <h3>&nbsp;<span style="font-weight: 400">How do I place a bootable ISO on a USB drive?</span></h3> 
       <p><span style="font-weight: 400">(</span><a href="https://superuser.com/questions/66948/how-do-i-place-a-bootable-iso-on-a-usb-drive"><span style="font-weight: 400">Question ID 66948</span></a><span style="font-weight: 400">)</span></p> 
       <p>This is a popular question with many similar or overlapping questions. Browsing the various linked and related questions I noticed the information on this subject is limited and fragmented. Most answers are simple software recommendations and do not touch upon any underlying issues. This article is meant to help fellow superusers appreciate the assumptions we take for granted and serve as a springboard for those that need to dig deeper to solve their problem.</p> 
       <p><span style="font-weight: 400">Most people who are looking to place an ISO on a USB drive want to&nbsp;</span><span style="font-weight: 400">put </span><i><span style="font-weight: 400">any</span></i><span style="font-weight: 400"> ISO on a USB drive and make the drive bootable (a</span>lso known as making a live USB or installer USB from a binary image file). Examples include:</p> 
       <ul> 
        <li>Windows installer / Windows recovery</li> 
        <li>Some other&nbsp;rescue media&nbsp;needed to restore a&nbsp;backup&nbsp;(e.g. <a href="http://www.sevenforums.com/tutorials/218920-macrium-reflect-create-bootable-rescue-usb-drive.html">Macrium reflect</a>)</li> 
        <li>Linux live image, possibly <a href="https://www.maketecheasier.com/persistent-live-usb-vs-full-install-usb/">persistent</a></li> 
        <li>A full OS install onto a USB</li> 
        <li>does <strong>not</strong> include useless&nbsp;ISOs,&nbsp;e.g. non-data like those filled with music or video.</li> 
       </ul> 
       <p><span style="font-weight: 400">This is </span><b><i>not </i></b><span style="font-weight: 400">necessarily&nbsp;the same as:</span></p> 
       <ul> 
        <li style="font-weight: 400"><span style="font-weight: 400">putting an ISO on a bootable USB drive </span></li> 
        <li style="font-weight: 400"><span style="font-weight: 400">copying an ISO file or its filesystem contents onto a formatted USB drive</span></li> 
       </ul> 
       <p><span style="font-weight: 400">The problem we solve with today’s post is: “Given an ISO and a non-bootable USB drive, how do I create a bootable USB?”</span></p> 
       <h3><strong>What is an ISO anyway?</strong></h3> 
       <p><span style="font-weight: 400">In the broadest sense it is a binary image (a.k.a. carbon copy, snapshot or raw data backup) of the storage sectors of a block-device such as a DVD, USB drive, micro-sd card or HDD. The ISO format has no standard (ironically), so there is no pre-defined header (if any), nor any restriction on what filesystems may be contained. These image files are </span><i><span style="font-weight: 400">not </span></i><span style="font-weight: 400">related to graphical/visual image files like JPEGs. <em><strong>But</strong> </em>in common usage an ISO is expected to consist of a single standard filesystem, like the&nbsp;ISO 9660 filesystem.&nbsp;</span><span style="font-weight: 400">Here is another good definition (from <a href="http://www.magiciso.com/tutorials/miso-whatiso.htm">MagicISO </a>website):&nbsp;</span></p> 
       <blockquote>
        Going by the more restrictive definition, an “ISO” is created by copying an entire disc, from sector 0&nbsp;to the end, into a file. […]&nbsp;it isn’t possible to store anything but a single data track in this fashion. Audio tracks, mixed-mode discs, CD+G, multisession, and other fancy formats can’t be represented. [this specifically refers to discs with an ISO 9660 filesystem, as a&nbsp;
        <em>convention</em>]
       </blockquote> 
       <p>In case you’re wondering about tracks, they’re basically partitions–so obviously a single filesystem can not have partitions. Of course an image can contain multiple tracks/partitions, but if we refer to such an image as an ISO we are using the loose definition (and probably making some developer angry). So an ISO can be any data image, but by <strong>convention</strong> we mean the entire&nbsp;user data* from an optical disk, which is commonly formatted with a <strong>single filesystem</strong>. So your ISO will <em>probably</em> conform to this more restrictive definition.</p> 
       <p><span style="font-weight: 400">The name originated from CD images that commonly used ISO 9660 to store data. Nowadays most uses assume an ISO refers to an image from an optical disk like CD or DVD (even though in DVD the UDF filesystem is more common) . Even <a href="http://wiki.osdev.org/Mkisofs">tools</a>&nbsp;that can create an ISO from a HDD folder probably implicitly convert to an optical media filesystem like ISO 9660. For this reason it may be better for software to use a different extension like “.IMG” to denote non-optical image files.</span></p> 
       <p>*With CDs a raw sector is 2352 bytes, but only 2048 bytes are used for data in formatted storage. The rest is used for error correction related stuff. This is why you might see corrupted files if you extract a true raw image from a CD and try to mount it–files are injected with junk bits. DVDs don’t make this distinction, their sectors only have 2048 bytes of user data. So in the case of CDs with filesystems an ISO is normally not a true sector-by-sector carbon copy.</p> 
       <blockquote>
        These days it’s not altogether uncommon to see messages about “making an ISO” of an audio CD, which makes no sense at all. (
        <a href="http://www.magiciso.com/tutorials/miso-whatiso.htm">MagicISO FAQ</a>)
       </blockquote> 
       <h3><strong>Aha, a copy ‘sectors’ from a ‘block device’…</strong></h3> 
       <p>A&nbsp;block device is just jargon for storage media, or ‘drive’ if you like. From a programmer’s point of view storage is a one dimensional array (row/line) of bits. But at the hardware level you can’t address each bit individually. The smallest ‘word’ size for reading and writing is called the sector size. Filesystems are implemented to improve aspects like speed or security (e.g. we’d like for&nbsp;files to have names instead of a sector number like 0x0000008AF1). It has been a while since I attempted graphics:</p> 
       <div id="attachment_5634" style="width: 693px" class="wp-caption alignleft">
        <a href="http://blog.superuser.com/files/2016/01/block_device3.png"><img class="wp-image-5634 size-full" src="http://blog.superuser.com/files/2016/01/block_device3.png" alt="test" width="683" height="204"></a>
        <p class="wp-caption-text">Figure 1 Block Device: The smallest addressable unit in a block device is called a sector. CDs and DVDs typically have a sector size of 2048 bytes. Filesystems (managed by the OS) may group multiple sectors into blocks. E.g. 4 sectors in the physical addressing space may constitute 1 block address in the logical (filesystem) addressing space–see image. The same data may occupy different sectors depending on the filesystem type. In the case of ISO 9660 a block is normally also 2048 bytes (but this value isn’t demanded by&nbsp;the standard).</p>
       </div> 
       <p>So files occupy <em>at least</em>&nbsp;1 sector. Images (ISOs in the loose definition) are created by copying all the bits in all the sectors. In case the data occupies less than the whole sector (e.g small files)&nbsp;the other bits are still copied instead of being zeroed out, because the filesystem may require them. On a sidenote, in my opinion file based images (like WIM used in Windows backups) do not fit the definition of “image”.&nbsp;But that is a discussion for another article.</p> 
       <p>The size of a sector depends on the hardware. Careful when burning an image your software does not make any wrong assumptions. For example copying every two original sectors into one new one might mess up the filesystem.</p> 
       <blockquote>
        Beginning in late 2009, accelerating in 2010 and hitting mainstream in 2011, hard drive companies are migrating away from the legacy sector size of 512 bytes to a larger, more efficient sector size of 4096 bytes, generally referred to as 4K sectors, and now referred to as the Advanced Format by IDEMA (The International Disk Drive Equipment and Materials Association). (
        <a href="http://www.seagate.com/gb/en/tech-insights/advanced-format-4k-sector-hard-drives-master-ti/">seagate</a>)
       </blockquote> 
       <div id="attachment_5648" style="width: 310px" class="wp-caption alignleft">
        <a href="http://blog.superuser.com/files/2016/01/manage_images.png"><img class="size-medium wp-image-5648" src="http://blog.superuser.com/files/2016/01/manage_images-300x143.png" alt="You can manage disk images with tools like these (bit outdated)." width="300" height="143"></a>
        <p class="wp-caption-text">You can manage disk images with tools like these.</p>
       </div> 
       <p><span style="font-weight: 400">Tools that can create an ISO from optical media are common. Most disk burning software will also have this functionality. Though it seems much harder to find software that will make a true image of non-optical media (like a HDD or USB drive). Most backup software (Acronis, Paragon, Norton, etc.) &nbsp;will do this, but they may use proprietary formats and/or compression.&nbsp;</span></p> 
       <p><span style="font-weight: 400">It is also possible to image individual partitions and even files on a hard drive. Mounting or file exploring software will need to recognize the filesystem before it becomes usable. Ok so, enough about images, how do we&nbsp;use&nbsp;some&nbsp;ISO with a&nbsp;flash drive?</span></p> 
       <blockquote>
        *A sidenote for Windows users: If you want to quickly get an ISO from your DVD or explore the files of one, 7zip might be all you need. If you want to make an ISO from folders you’ll need to try something more advanced like imgburn–it lets you add/load a custom bootsector. Surprisingly few utilities support making ISOs from folders.&nbsp;Note that only adding a bootsector will not re-create the original ISO if hidden settings were lost.
       </blockquote> 
       <p><strong>Bootable vs Non-bootable</strong></p> 
       <p><span style="font-weight: 400">The question as it is now specifically refers to bootable ISOs as opposed to non-bootable ISOs. Is there such a distinction, and if so, what’s the difference? Well, bootable ISOs contain ‘invisible’ data called &nbsp;boot-sectors. Now this is where it gets messy because many people don’t know what kind of firmware they have (BIOS or UEFI) and it determines the bootcode requirements.</span></p> 
       <p><em>UEFI</em></p> 
       <p><span style="font-weight: 400">UEFI systems don’t need this invisible data to boot, provided your files include .efi files in the <a href="https://www.thomas-krenn.com/en/wiki/Creating_Windows_UEFI_Boot-Stick_in_Windows">right place</a> on your USB drive. The drive should also be marked active and be formatted with the FAT32 filesystem. Simple. This is why some cases of copy and pasting the ISO contents just work, while others do the same steps and fail miserably on their BIOS systems. In certain&nbsp;cases the USB drive has an MBR instead of a GPT and the UEFI goes into BIOS mode without trying to look for the .efi files. To fix this bad behavior either change a setting in the UEFI or change the drive to a GPT type partitioning.</span></p> 
       <p><em>BIOS</em></p> 
       <p>So technically an ISO with efi files might be bootable, but I will reserve the term “bootable ISO” for those images with invisible boot files. For the BIOS case you will need this invisible data (i.e. a bootable ISO) unless your USB drive was previously formatted with compatible boot code, in which case you can just replace the visible filesystem contents from the ISO–copy paste.&nbsp;Additionally, in some rare cases the BIOS might be&nbsp;smart enough to boot without a sector like UEFI.</p> 
       <p><span style="font-weight: 400">There are normally two sectors involved in loading the operating system, the&nbsp;<a href="https://en.wikipedia.org/wiki/Master_boot_record">MBR </a>and the <a href="https://en.wikipedia.org/wiki/Volume_boot_record">VBR</a>. This implies a&nbsp;BIOS system can’t&nbsp;boot from a GPT style drive. To get this invisible data from your ISO to your USB you need to burn the image, not just copy the filesystem (visible) files. If you don’t know if your ISO is bootable, burn it and test it, or maybe check the <a href="https://superuser.com/questions/683210/how-do-i-determine-if-an-iso-is-a-hybrid">first sector</a>.</span></p> 
       <blockquote>
        “ The location and size of the boot sector (perhaps corresponding to a logical disk sector) is specified by the design of the computing platform.”
       </blockquote> 
       <p><span style="font-weight: 400">— Wikipedia (</span><a href="https://en.wikipedia.org/wiki/Boot_sector"><span style="font-weight: 400">Boot_sector</span></a><span style="font-weight: 400">)</span></p> 
       <p><span style="font-weight: 400">So it’s a few Kb somewhere that the CPU loads to do the bootstrapping process. You can’t access this data with a file explorer, because&nbsp;it is&nbsp;not part of the filesystem proper. A good ISO image editor should let you modify&nbsp;all the bytes, possibly overwriting them with custom sub-images.</span></p> 
       <p><span style="font-weight: 400">Now the basic theory is out of the way, perhaps&nbsp;a better question is: what about all those free tools that take an ISO and give back a bootable USB?&nbsp;I suppose some tools may&nbsp;add the invisible stuff for you if you have&nbsp;a non-bootable ISO. Or they might choose between MBR and GPT for you. Or you might be having trouble finding software you like that burns your ISO to your drive. They just make life a bit easier. But they might not consider all the&nbsp;variables I’ve mentioned.</span></p> 
       <p><span style="font-weight: 400">The most important thing to remember is a USB drive can be made bootable independently of the data. The following general steps can be taken to create our live/installer USB:</span></p> 
       <ol> 
        <li>If you don’t know if your ISO is bootable, burn and test it. If that fails then wipe and:</li> 
        <li><span style="font-weight: 400">Make USB bootable by adding bootsector(s), formatting may wipe the drive. A common tool is <a href="https://technet.microsoft.com/en-us/library/cc749177(v=ws.10).aspx">bootsect.exe</a>.&nbsp;This is needed for BIOS systems. This process (particularly the VBR) is often OS specific, but many non-windows solutions use GRUB or Syslinux (<a href="https://wiki.archlinux.org/index.php/Boot_loaders">link</a>). There are many tutorials on the web about how to install one of these bootsectors. But before you get stuck in, invest some more time into finding a working image file, most developers distribute them.&nbsp;</span></li> 
        <li><span style="font-weight: 400">Add </span><i><span style="font-weight: 400">compatible</span></i><span style="font-weight: 400"> files (e.g. linux distro, windows installer, rescue program). You can just copy the files contained by the ISO to your USB drive. Depending on your target operating system your folder structure may need to be in&nbsp;a specific format. In <a href="http://www.pendrivelinux.com/install-grub2-on-usb-from-ubuntu-linux/">some cases</a> you are allowed to copy the whole ISO.</span></li> 
       </ol> 
       <p><span style="font-weight: 400">No-one wants to burn a DVD twice, even if they could. But once you have a bootable USB drive you can simply copy the data or folders to it. As long as your invisible boot-sector is valid for whatever you want to use you can just swap files. I believe Vista and up use compatible boot code for BIOS and UEFI. An alternative route is to modify the ISO directly and then burn it all at once.&nbsp;</span></p> 
       <p><span style="font-weight: 400">&nbsp;Though we agreed to follow the ISO convention, ISO hybrids exist. They&nbsp;already contain the MBR code the flash drive needs while also keeping the 9660 filesystem intact. Rarely are USB-only images distributed (unfortunately).</span></p> 
       <div id="attachment_5635" style="width: 310px" class="wp-caption alignnone">
        <a href="http://blog.superuser.com/files/2016/01/optical_disk_layout.png"><img class="size-medium wp-image-5635" src="http://blog.superuser.com/files/2016/01/optical_disk_layout-300x140.png" alt="The above shows how a possible hybrid ISO can host both types of bootsector (optical and HDD). Only the green area is viewable to a typical user when mounting the partition." width="300" height="140"></a>
        <p class="wp-caption-text">The above shows how a possible hybrid ISO can host both types of bootsector (optical and HDD). Only the green area is viewable to a typical user when mounting the partition.</p>
       </div> 
       <p><span style="font-weight: 400">There are many tools like the official Microsoft tool (probably uses bootsect), Rufus, YUMI (a.k.a pendrivelinux) and Unetbootin. I’m not sure how these programs do everything, but&nbsp;I believe most software will require a bootable ISO to generate the bootable USB sector. Perhaps they translate the boot volume descriptor (optical disk filesystem) into an MBR compatible with whatever filesystem they use to format the drive (probably FAT32, or maybe they leave it in 9660).</span></p> 
       <p><b>So what if you have a bootable ISO, but it has&nbsp;become unbootable?</b></p> 
       <p><span style="font-weight: 400">Normally non-bootable ISOs are useless for the purposes of live/installer USBs (e.g. a DVD video ISO)–unless you somehow accidentally end up with a non-bootable ISO that <em>should</em> have been bootable.</span></p> 
       <p><span style="font-weight: 400">For example, if you take a bootable DVD you can create an ISO from it both with or without the bootable property, depending on how you do it. In fact you can save any file or folder in ISO format, because there is no standard–it is just a filesystem snapshot. It is the settings used in the ISO creation process that determine if the produced ISO is bootable or not. </span></p> 
       <p><span style="font-weight: 400">People often end up with non-bootable ISOs when they try to customize their ISO or create an ISO from a folder. The bootable property is lost when extracting an ISO and then repacking it without setting ‘make bootable’ options. Be sure to copy your bootsector.bin from the original ISO.</span></p> 
       <p><span style="font-weight: 400">Now, let’s say you have a folder with files you want to turn into a bootable USB. You can either manually make the USB bootable, but this may be tricky for some&nbsp;images/systems. Or you can try and turn the folder into a bootable ISO, which you then supply to some one-click tool to create the live/installer USB (you’d need to find/create a bootsector first–and no guarantee it will work). Lastly there may be a niche tool that converts your customized folder directly to a working official ISO (like WAIK or&nbsp;RT se7en Lite for Windows images).</span></p> 
       <p><span style="font-weight: 400">Note the lack of 2&nbsp;options: there doesn’t seem to be software that accepts a folder directly, probably due to the lack of bootsector info, though it should work for modern UEFI systems. And there doesn’t seem to be a tool that can make a USB bootable for a given image distribution (i.e. just do step 2&nbsp;in a few&nbsp;clicks). I’d be happy to hear information to the contrary!</span></p> 
       <h3><strong>Conclusion</strong></h3> 
       <p><span style="font-weight: 400">Many one-click tools do require an ISO (preferably a bootable one) to generate a bootable USB. Just remember that an ISO may be&nbsp;more than just a file archive, and the underlying filesystems may vary. Get the right ISO and any burning software will copy all the sectors to your USB drive, it shouldn’t be hard. And don’t be worried&nbsp;if the extension is something like .img or .bin, in many cases an image is an image!</span></p>
