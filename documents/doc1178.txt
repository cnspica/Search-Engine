            <div class="post-text" itemprop="text"> 
             <p>I think I discovered a bug in the new navigation system - have a look:</p> 
             <p><a href="http://i.stack.imgur.com/zobGI.gif" rel="nofollow"><img src="http://i.stack.imgur.com/zobGI.gif" alt="gif of the reproduction"></a></p> 
             <p>If I have a navigation tab open and filter for two or more tags (set to unanswered, any) and then reload the page, a new tab is being created. The tag that was open before was saved and had no unsaved changes to it. It does not matter, whether it is set to <em>any</em> or <em>all</em>, neither is it important whether there are unsaved changes or not.</p> 
             <p>Here is the mandatory freehand circle, displaying what should not happen:</p> 
             <p><a href="http://i.stack.imgur.com/aKrqv.png" rel="nofollow"><img src="http://i.stack.imgur.com/aKrqv.png" alt="freehand circles, yay"></a></p> 
             <p>This does not happen if the search contains only one tag.</p> 
             <p>I reproduced this on three different machines, all running different browsers:</p> 
             <ul> 
              <li>Safari 9.1 11601.5.17.1 (MacOS X El Capitan)</li> 
              <li>IE11 11.0.9600.18204 (Windows 7)</li> 
              <li>FF 46.0.1 (Windows 10)</li> 
             </ul> 
             <p>I also just reproduced this on my Android tablet, running Chrome 50.0.2661.89</p>
