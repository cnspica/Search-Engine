      <h1 class="blog-title"><a href="http://blog.superuser.com/2013/06/13/windows-8-on-a-vhd-trying-windows-without-the-risk/">Windows 8 on a VHD – Trying windows without the risk</a></h1> 
      <div class="meta-info">
       <span class="postedDate">June 13, 2013</span> by 
       <a href="http://blog.superuser.com/author/journeymangeek/" title="Posts by journeymangeek" rel="author">journeymangeek</a>. 
       <a href="http://blog.superuser.com/2013/06/13/windows-8-on-a-vhd-trying-windows-without-the-risk/#comments">3 comments</a> 
      </div> 
      <div class="post-text"> 
       <p>I’m currently waiting on building a new system (which will run windows 8) but <a href="http://win8challenge.com/">I wanted in on the windows contest </a>. I didn’t really want to use one of my windows 8 licences on my old system, so I figured I’d run the enterprice evaluation version on a VHD, which I could discard to get back my current system to its previous state once the new system is built. I found a great guide by <a href="http://blogs.technet.com/b/haroldwong/archive/2012/08/18/how-to-create-windows-8-vhd-for-boot-to-vhd-using-simple-easy-to-follow-steps.aspx">Harold Wong on technet</a>, and while I was working finishing up the post this guide is based off , discovered you could install to a hard drive using the same method, found that <a href="http://superuser.com/questions/490476/how-can-i-install-windows-8-on-my-hdd-without-the-use-of-a-usb-or-dvd/490477#490477">r.tanner.f </a>had used a similar method for an install on a actual drive. I used a windows 7 system to set this up and run it on, and you will probably find it easier if you use the WAIK from the same architecture as this system. This is pretty much like wubi – allowing me to run a seperate copy of windows without repartitioning my drive.</p> 
       <p><span id="more-5239"></span></p> 
       <h2>Prerequisites:</h2> 
       <ul> 
        <li>Windows 8 install disk – I went with the <a href="http://msdn.microsoft.com/en-us/evalcenter/jj554510.aspx">enterprise evaluation disk</a>, You could probably do this with a regular installer with a key as well</li> 
        <li>imagex from <a href="http://www.microsoft.com/en-us/download/details.aspx?id=5753">WAIK</a> , I had it from a previous experiment</li> 
        <li><a href="http://neosmart.net/EasyBCD/">easybcd</a> , bcdboot (its in system32 and part of a default windows system), <a href="http://www.boyans.net/">visualbcd</a> or some other bcd editor</li> 
       </ul> 
       <h2>Preparing a VHD</h2> 
       <p>You can create a VHD from disk manager – hit start, and search for disk management. Select Create and format hard disk partitions (or go to control panel -&gt; computer management -&gt; disk management) <img class="alignnone" alt="" src="http://i.stack.imgur.com/rHzQU.png" width="407" height="504"></p> 
       <p>Creating a VHD is very simple, select action, and Create VHD</p> 
       <p><img class="alignnone" alt="" src="http://i.stack.imgur.com/fX8zv.png" width="635" height="458"></p> 
       <p>Set up VHD to taste, I’ve selected a large dynamic disk on the partition I usually store VM images on.</p> 
       <p><img class="alignnone" alt="" src="http://i.stack.imgur.com/FnWi8.png"></p> 
       <p>This should result in a new ‘disk’ you can use like any other disk. Rightclick on the disk name, and select initialise, then rightclick on the partition on the right side to format. Take note of the new drive latter – here its H:</p> 
       <p><img class="alignnone" alt="" src="http://i.stack.imgur.com/5E5GU.png" width="1425" height="625"></p> 
       <hr> 
       <h2>Installation process</h2> 
       <p>At this point, we pretty much can treat the VHD the very same way we’d treat a regular hard drive. You’ll want to extract WAIK and the install ISOs to separate folders, or mount them. Take note of the path of install.wim on the windows 8 install cd – this is usually in <code>\install\sources\</code> relative to the root of the windows 8 files</p> 
       <p>Copy imagex and (if you’re using it bcdboot) to a convenient location. Start up a command prompt with admin privileges</p> 
       <p><img class="alignnone" alt="" src="http://i.stack.imgur.com/IgXlo.png"></p> 
       <p>Run imagex from where you saved it – <code>imagex /apply /path/to/install.wim 1 h:\</code> where h: is the vhd’s drive letter. This took me around 10 minutes</p> 
       <p><img class="alignnone" alt="" src="http://i.stack.imgur.com/mxMfw.png" width="687" height="351"></p> 
       <p>There’s two options to make the image bootable, firstly, to run bcdboot from WAIK. I went with easybcd instead, since I didn’t have bcdboot on hand. Select add new entry, and set things as follows in the windows <em>not</em> the VHD tab. We’re treating the VHD like a normal disk.</p> 
       <p><img class="alignnone" alt="" src="http://i.stack.imgur.com/4SGeu.png" width="588" height="495"></p> 
       <p>Now reboot, and select windows 8 from the windows boot manager, and you should be set. The second stage installer will start, and you can continue the installation as per normal. If you don’t want the install of windows 8 any more simply delete the boot entry and VHD</p> 
       <p>You can also easily backup the VHD by detaching, then reattaching it. Dosen’t get any easier than that <img src="http://blog.superuser.com/wp-includes/images/smilies/simple-smile.png" alt=":)" class="wp-smiley" style="height: 1em; max-height: 1em;"></p> 
       <p>&nbsp;</p>
