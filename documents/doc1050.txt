             <p>This is an unofficial list of new features and various changes to Stack Overflow and the Stack Exchange network.</p> 
             <p>It is maintained by the community, but a Stack Exchange developer changes the Accepted Answer to ensure that the latest changes remain on top (given default user settings).</p> 
             <p>To see the list in reverse chronological order (latest entry on top), <strong><a href="http://meta.stackexchange.com/questions/59445/recent-feature-changes-to-stack-exchange?answertab=active#tab-top">sort Answers by latest activity first</a></strong>.</p> 
             <p>For those adding new features here: please post only feature changes that affect the whole network and have some global impact. E.g. bug fixes don't belong here, as well as mobile app new versions.</p>

             <h3>May 2016</h3> 
             <ul> 
              <li><p>2016-05-04: <a href="http://meta.stackexchange.com/a/279201/165773">PostHistory record added when Community bumps a post</a> (there's no record prior to May 4th, 2016).</p></li> 
              <li><p>2016-05-03: <a href="http://meta.stackoverflow.com/a/322265/839601">The new-user ask limit is now one question every <strong>40</strong> minutes, <em>network-wide.</em></a> That means only 1 question every 40 minutes on Stack Overflow, but also means you'd need to wait 40 minutes after asking a question on, say, Woodworking to ask a related question on Crafting or Home Improvement.</p></li> 
             </ul> 
             <h3>April 2016</h3> 
             <ul> 
              <li>2016-04-06: <a href="http://meta.stackexchange.com/a/278000/165773">Cross-site posting rate limits</a> apply to new users on every site forcing a waiting period between posting a question or answer after having previously posted one <em>anywhere else on the network</em>.</li> 
             </ul> 
             <h3>March 2016</h3> 
             <ul> 
              <li><p>2016-03-31: Users can ask on <a href="/questions/tagged/per-site-meta" class="post-tag" title="show questions tagged 'per-site-meta'" rel="tag">per-site-meta</a> about their <em>own</em> questions <a href="http://meta.stackexchange.com/a/277727/165773">even when their reputation is less than 5</a>.</p></li> 
              <li><p>2016-03-23: Users can <a href="http://meta.stackexchange.com/a/275796/165773">see review tasks linked in the timeline once they're not currently live (completed or invalidated)</a>.</p></li> 
              <li><p>2016-03-21: <a href="http://meta.stackoverflow.com/a/319465/839601">Common link shorteners are blacklisted in posts</a> at Stack Overflow.</p></li> 
              <li><p>2016-03-15: 15K users can't protect a question <a href="http://meta.stackexchange.com/a/276978/162102">if it has no answers from new users</a>.</p></li> 
              <li><p>2016-03-07: Hidden Communities <a href="http://meta.stackexchange.com/a/276581/152859">no longer appear in the Network Profile accounts list</a>. (Moderators can still see them.)</p></li> 
              <li><p>2016-03-02: <a href="http://meta.stackexchange.com/a/276380/152859">New moderator-only page is now available</a>, called Duplicate Comments, allowing moderators to detect and handle users posting series of identical comments to spam/abuse the site.</p></li> 
             </ul> 
             <h3>February 2016</h3> 
             <ul> 
              <li><p>2016-02-15: Users can now <a href="http://meta.stackexchange.com/a/275694/162102">choose which site to sync the network profile's "about" text from</a>, and this text is no longer automatically refreshed every day.</p></li> 
              <li><p>2016-02-15: Users can now <a href="http://meta.stackexchange.com/a/275687">choose whether to see more ads if they've reached the "Reduce Ads" privilege</a>.</p></li> 
              <li><p>2016-02-07: Moderators can now <a href="http://meta.stackexchange.com/a/274274">redact a revision</a> of a post. (Requires two moderators, one to perform the edit and another one to approve.)</p></li> 
              <li><p>2016-02-01: Post deletion is now included in the heartbeat, and will trigger a <a href="http://i.stack.imgur.com/TNq3U.png" rel="nofollow">popover message</a> indicating that the post has been deleted while you're viewing it. The flag dialog will also prevent you from opening it if the post's state has changed (<a href="http://i.stack.imgur.com/3IBeC.png" rel="nofollow">deleted</a> or <a href="http://i.stack.imgur.com/4Tf5X.png" rel="nofollow">closed</a>) since you loaded the page.</p></li> 
             </ul> 
             <h3>January 2016</h3> 
             <ul> 
              <li><p>2016-01-20: <a href="http://meta.stackexchange.com/a/273402/165773">Spam and offensive flags on a post will be preserved when the post's author deletes it</a>.</p></li> 
              <li><p>2016-01-19: Moderators can now <a href="http://meta.stackexchange.com/a/273359">change the edit summary of revisions</a>.</p></li> 
              <li><p>2016-01-18: Chat room owners and moderators are <a href="http://meta.stackexchange.com/a/273268">now notified when a user is kicked</a>.</p></li> 
              <li><p>2016-01-12: Sidebar width has been <a href="http://meta.stackexchange.com/questions/272563/were-standardizing-the-sidebar-width-at-300px-on-all-sites">standardized at 300 pixels on all sites</a>.</p></li> 
              <li><p>2016-01-06: <a href="http://meta.stackexchange.com/questions/260153/global-auth-is-dead-long-live-universal-login">Global authentication is replaced with Universal login</a>, network wide. In a nutshell, it means that once you have an account in one site, all you need to do to login into another site is click "join this community", without re-entering any credentials.</p></li> 
             </ul>

             <h3>December 2011</h3> 
             <ul> 
              <li><p>2011-12-28: <a href="http://meta.stackexchange.com/a/117273">There's</a> a tag-wiki-specific "not helpful... does not follow any of our tag wiki guidelines" suggested-edit rejection reason.</p></li> 
              <li><p>2011-12-22: The <code>/reputation</code> page now shows the reputation earned from suggested edits.</p></li> 
              <li><p>2011-12-21: Moderators (and the users themselves) can now see <a href="http://meta.stackexchange.com/q/116793">history of past user names</a> (for users that have ever changed their name)</p></li> 
             </ul> 
             <h3>November 2011</h3> 
             <ul> 
              <li><p>2011-11-28: The newly redesigned profile pages have been pushed network-wide.</p></li> 
              <li><p>2011-11-28: Spoiler text is now <a href="http://meta.stackexchange.com/questions/89552/can-we-please-remove-spoilered-text-from-the-summary/113889#113889">hidden in Questions pages and on front page tooltips</a>.</p></li> 
              <li><p>2011-11-23: New <a href="http://meta.stackexchange.com/questions/63841/show-a-short-link-for-answers-too-not-just-questions">short link for answers</a> in the form of <a href="http://meta.stackexchange.com/a/59446"><code>stackoverflow.com/a/[answer_id]</code></a>. These may also contain a user ID for tracking purposes, but sharing them <a href="http://meta.stackexchange.com/questions/114607/do-shared-link-to-answers-also-count-for-announcer-booster-publicist-badges/114608#114608">does not award publicity badges</a>.</p></li> 
              <li><p>2011-11-22: The new-look profiles have been rolled out to <a href="http://serverfault.com">Server Fault</a> and <a href="http://superuser.com">Super User</a>, but not the site metas. Also other sites including EL&amp;U, WepApps, Gaming and Ubuntu.</p></li> 
              <li><p>2011-11-15: <a href="http://meta.stackexchange.com/questions/112151/new-user-page-live-on-meta">The profile page has been redesigned and the new style deployed to Meta only for testing</a>. </p></li> 
              <li><p>2011-11-08: The <a href="http://meta.stackexchange.com/questions/110698/new-bronze-badge-for-reviewing/111329#111329">Reviewer badge has been changed to a silver badge with the requirements increased to 1000 reviews and acting on at least 200</a>.</p></li> 
              <li><p>2011-11-07: nofollow <a href="http://meta.stackexchange.com/questions/111279/remove-nofollow-on-links-deemed-reputable">removed from</a> links in "reputable" posts.</p></li> 
              <li><p>2011-11-07: New bronze badge "<a href="http://meta.stackexchange.com/questions/110698/new-bronze-badge-for-reviewing/110700#110700">Reviewer</a>" for reviewing 300 posts and acting on over 60 of them.</p></li> 
              <li><p>2011-11-02: When reviewing a suggested edit, you can now <a href="http://meta.stackexchange.com/questions/110277/change-default-selection-in-edit-rejection-dialogue-box-from-copied-content-to/110811#110811">reject it with a custom reason</a>.</p></li> 
             </ul> 
             <h3>October 2011</h3> 
             <ul> 
              <li><p>2011-10-27: The review section now lets you <a href="http://meta.stackexchange.com/questions/75953/allow-marking-posts-as-reviewed-on-review-and-hide-them/110414#110414">mark posts as reviewed</a>. Doing so allows you to directly take actions on these posts (such as voting, flagging, commenting, …), and hides them from the review section.</p></li> 
              <li><p>2011-10-18: Stack Exchange <a href="http://meta.stackexchange.com/questions/109795/what-shortened-urls-are-available-through-s-tk">URL shortener s.tk launched</a>.</p></li> 
              <li><p>2011-10-13: <a href="http://meta.stackexchange.com/questions/108886/add-blog-highlights-to-stackexchange-newsletters">Blogs snippets added</a> to Stack Exchange newsletters.</p></li> 
              <li><p>2011-10-11: <a href="http://meta.stackexchange.com/questions/92602/wheres-the-summary-of-your-own-suggested-edits">"Suggestions" filter added to the activity tab</a> in user profiles, showing all the edits that member suggested and their status.</p></li> 
              <li><p>2011-10-08: Email notifications <a href="http://blog.stackoverflow.com/2011/10/improved-global-email-notifications/">overhauled and integrated with the global inbox</a>.</p></li> 
              <li><p>2011-10-07: When placing a bounty on question asked by other user, <a href="http://meta.stackexchange.com/a/132622/152859">you are getting a notification for new answers on that question as long as the bounty is active</a>.</p></li> 
             </ul> 
             <h3>September 2011</h3> 
             <ul> 
              <li><p>2011-09-30: When rejecting an edit you can now <a href="http://meta.stackexchange.com/questions/77875/pending-approvals-allow-for-adding-reasons-to-rejections/107930#107930">choose a pre-defined reason</a>.</p></li> 
              <li><p>2011-09-27: "Post Made Community Wiki" <a href="http://meta.stackexchange.com/questions/106569/could-the-made-community-wiki-by-entry-in-the-revision-list-show-the-actual">now showing the actual reason</a>.</p></li> 
              <li><p>2011-09-26: Links to comments on the profile page will now <a href="http://meta.stackexchange.com/questions/94486/comment-links-on-responses-tab-do-not-work-correctly-when-the-comment-is-hidden/107511#107511">always take you directly to the comment</a>, even if it would usually be hidden (currently only if the comment is on an answer).</p></li> 
              <li><p>2011-09-23: Moderators can add a <a href="http://meta.stackexchange.com/questions/102363/support-for-banners">notice to a post</a> that will display at the bottom of the post.</p></li> 
              <li><p>2011-09-23: You must be a <a href="http://meta.stackexchange.com/questions/107152/encouraging-users-to-create-an-account-and-keep-it/107163#107163">registered user to ask questions</a> on Stack Overflow. Unregistered users may still answer questions however.</p></li> 
              <li><p>2011-09-22: On Stack Overflow, <a href="http://meta.stackexchange.com/questions/107058/how-do-you-prevent-here-code-you-fix-questions">low-reputation users must enter a nominal ratio of non-code characters for each line of code</a> in a question before they can ask it. (This was also discussed during <a href="http://blog.stackoverflow.com/2011/09/se-podcast-19/">SE Podcast #19</a>.)</p></li> 
              <li><p>2011-09-22: Users are now <a href="http://meta.stackexchange.com/questions/105309/how-do-you-want-to-be-notified-of-unaccepted-answers">notified of unaccepted answers</a>.</p></li> 
              <li><p>2011-09-20: The minimum bounty on a question <a href="http://meta.stackexchange.com/questions/64824/clever-bounty-reputation-hack">you have answered</a> is now 100. </p></li> 
              <li><p>2011-09-20: Questions with an <a href="http://meta.stackexchange.com/questions/104914/add-title-uniqueness-to-the-heuristics-for-detecting-low-quality-questions">identical title</a> to an existing question can no longer be submitted.</p></li> 
              <li><p>2011-09-20: The featured tab now sorts with a weight that gives a <a href="http://meta.stackexchange.com/questions/105284/too-many-featured-questions">Colbert Bump</a> to higher bounties.</p></li> 
              <li><p>2011-09-20: <a href="http://meta.stackexchange.com/questions/103956/how-should-bounties-be-categorized">Bounty remarks</a> are now supported.</p></li> 
              <li><p>2011-09-20: The badges page now includes <a href="http://meta.stackexchange.com/questions/82145/improve-the-badge-page-to-filter-by-badge-type">filtering by type</a>.</p></li> 
              <li><p>2011-09-19: <code>@name</code> can now be <a href="http://meta.stackexchange.com/questions/53404/tab-name-completion-for-comments-please">autocompleted in comments</a> using the tab key, similar to chat.</p></li> 
              <li><p>2011-09-17: Flagging a post for moderator attention, and then casting the final <a href="http://meta.stackexchange.com/questions/106505/users-can-raise-their-own-flag-weight-o/106512#106512">deletion</a> or <a href="http://meta.stackexchange.com/questions/106002/flagging-a-post-before-submitting-the-5th-close-vote-always-results-in-flag-being">close</a> vote, no longer earns an increase in flag weight. </p></li> 
              <li><p>2011-09-16 Network profiles no longer use ugly GUIDs, but a pretty account id.</p></li> 
              <li><p>2011-09-16: Management and promotion of <a href="http://stackexchange.com/newsletters">newsletter subscriptions</a>.</p></li> 
              <li><p>2011-09-16: Locked posts now include <a href="http://meta.stackexchange.com/questions/106137/better-handling-of-locked-questions-and-answers">an explanation of why</a> they were locked (<a href="http://stackoverflow.com/questions/361271/best-free-controls-for-net">example</a>), and can be optionally configured to automatically unlock after a certain amount of time has passed.</p></li> 
              <li><p>2011-09-14: Email notifications for individual questions are no longer available. Instead, users can receive <a href="http://meta.stackexchange.com/q/106303">email notifications of unread inbox items</a>.</p></li> 
              <li><p>2011-09-14: Deleting a post that is the master of a closed duplicate will now provide a warning.</p></li> 
              <li><p>2011-09-14: An advanced search option <code>hasnotice:1</code> has been added to find posts with moderator notices.</p></li> 
              <li><p>2011-09-12: The revision diff list <a href="http://meta.stackexchange.com/questions/72645/could-the-change-from-an-edit-be-displayed-in-a-side-by-side-display/105971#105971">now offers a side-by-side diff in addition to the inline diff</a>.</p></li> 
              <li><p>2011-09-12 Declined flags will now <a href="http://meta.stackexchange.com/q/106039">include a reason</a> that is included in the user's Flagging Summary.</p></li> 
              <li><p>2011-09-12: You now have <a href="http://meta.stackexchange.com/questions/79015/now-that-20k-users-can-delete-answers-should-we-increase-the-number-of-daily-del">extra delete votes</a> for every 1k above 10k reputation (beta sites may differ), with an upper limit of 30 delete votes per day.</p></li> 
              <li><p>2011-09-12: <a href="http://meta.stackexchange.com/questions/105755/roll-out-10k-tools-gradually-between-3k-and-10k-reputation-for-users-who-have-sho/105870#105870">At 5k rep</a>, users now have full access to the suggested edit queue.</p></li> 
              <li><p>2011-09-11: <a href="http://meta.stackexchange.com/questions/105275/abusing-bounties-for-unlimited-profit/105639#105639">The minimum amount of rep that can be spent on a bounty now doubles for each successive bounty on the same question by the same user.</a> In other words, if the first bounty on a question starts at 50 rep, the next bounty on that question has a minimum rep of 100, the next 200, the next 400, the next (max).</p></li> 
              <li><p>2011-09-09: LinkedIn removed from sharing options</p></li> 
              <li><p>2011-09-06: Users can no longer single-handedly undelete their own posts if they were deleted by vote; they can only <a href="http://meta.stackexchange.com/questions/43942/can-the-owner-of-a-question-get-it-undeleted-with-his-vote-alone-and-also-could/105300#105300">vote to undelete</a> like any other user.</p></li> 
              <li><p>2011-09-06: The sharing buttons are now shown on the trilogy in addition to the SE 2.0 sites.</p></li> 
              <li><p>2011-09-02: <a href="http://meta.stackexchange.com/questions/54315/why-cant-a-user-have-simultaneous-bounties-any-more/104813#104813">Up to 3 concurrent bounties</a> are allowed per user now.</p></li> 
              <li><p>2011-09-02: Voting to close as a duplicate now lists questions that were <a href="http://meta.stackexchange.com/questions/77103/improve-tools-for-closing-as-duplicate/104849#104849">previously used when closing related questions</a>.</p></li> 
             </ul> 
             <h3>August 2011</h3> 
             <ul> 
              <li><p>2011-08-27: Avatars now have a <a href="http://meta.stackexchange.com/questions/104810/how-does-the-user-card-popup-work">pop-up with user info</a> which expands when hovered over. (This was first tested on Meta sites and is now deployed everywhere.)</p></li> 
              <li><p>2011-08-24: <a href="http://facebook.stackoverflow.com">facebook.stackoverflow.com</a> "mini-site", as <a href="http://blog.stackoverflow.com/2011/08/facebook-stackoverflow/">the new official developer support channel</a> for all Facebook developers. When using a Facebook account to log in, this also pulls your Facebook profile picture to be used as your avatar, <a href="http://meta.stackexchange.com/questions/103734/my-gravatar-is-gone-after-adding-facebook-to-my-account/103753#103753">but you can revert that</a>.</p></li> 
              <li><p>2011-08-24: New bronze badge "<a href="http://stackoverflow.com/badges/1306/analytical">Analytical</a>" for visiting every section of the <a href="http://stackoverflow.com/faq">FAQ</a>.</p></li> 
              <li><p>2011-08-22: New gold badge "<a href="http://stackoverflow.com/badges/1298/marshall">Marshal</a>" for achieving a flag-weight of 749.</p></li> 
              <li><p>2011-08-17: Tools and Review <a href="http://meta.stackexchange.com/questions/102457/put-the-review-link-back-in-its-proper-place-next-to-tools-in-the-top-navigat/102501#102501">got separate category tabs</a> (on the left) at 10k tools; the <a href="/tools">/tools</a> link in the top bar remembers the last selection.</p></li> 
              <li><p>2011-08-16: The "links" tab on <code>/tools</code> is gone and now those links are inside other pages like "stats", "close", "delete".</p></li> 
              <li><p>2011-08-15: <a href="http://meta.stackexchange.com/badges/158/excavator">Excavator</a> and <a href="http://meta.stackexchange.com/badges/157/archaeologist">Archaeologist</a> badges added for editing posts that were inactive for 6 months.</p></li> 
              <li><p>2011-08-09: <a href="http://meta.stackexchange.com/a/101450/152859">Descriptive error message is now displayed</a> upon image upload failures.</p></li> 
              <li><p>2011-08-09: <a href="http://blog.stackoverflow.com/2011/08/improved-tagging/">Improved Tag auto-complete</a> to include tag wiki excerpt.</p></li> 
              <li><p>2011-08-09: Control-Enter in search box <a href="http://meta.stackexchange.com/q/101422">now opens results</a> in a new tab/window.</p></li> 
              <li><p>2011-08-08: The search text box now <a href="http://meta.stackexchange.com/questions/101387/please-tone-down-or-remove-the-auto-expansion-of-the-search-box">auto-expands to the left</a> when focused, temporarily hiding the top-bar links.</p></li> 
              <li><p>2011-08-06: <a href="http://meta.stackexchange.com/questions/90612/should-there-be-a-proofreader-badge/101171#101171">Proofreader badge</a> for having reviewed 100 suggested edits.</p></li> 
              <li><p>2011-08-02: In the 10k tools, <a href="http://meta.stackexchange.com/questions/100416/please-make-the-required-deletion-votes-count-visible-in-the-10k-tools/100442#100442">the <code>delete</code> tab now shows remaining needed vote counts</a>.</p></li> 
              <li><p>2011-08-01: Titles that appear to start with a tag now have the tag <a href="http://meta.stackexchange.com/questions/70498/automatic-removal-of-bracketed-tags-from-question-titles/100492#100492">stripped out</a>.</p></li> 
             </ul> 
             <h3>July 2011</h3> 
             <ul> 
              <li><p>2011-07-19: Comments containing multiple <code>@name</code> <a href="http://meta.math.stackexchange.com/questions/2094/should-we-discourage-the-user-of-multiple-redundant-username-pings/2615#2615">are blocked now</a> unless they contain a backtick.</p></li> 
              <li><p>2011-07-18: Trivial answers containing a link to another question in the network <a href="http://meta.stackexchange.com/questions/98950/how-come-my-answer-appears-as-a-comment/98955#98955">are automatically converted to comments on the question</a>.</p></li> 
              <li><p>2011-07-16: <a href="http://blog.stackoverflow.com/2011/07/mobile-stack-exchange/">Mobile theme</a> for Stack Exchange sites launched. Users can switch back and forth from the mobile theme using the 'mobile' link at the bottom of every page.</p></li> 
              <li><p>2011-07-15: Anonymous and low rep (&lt;15) users <a href="http://meta.stackexchange.com/questions/98630/anonymous-user-feedback-now-in-testing">now see a "give feedback" link</a> where logged in users usually see "add comment". When moused over, this shows "Was this post useful to you [yes] [no]". This is just for testing purposes right now - it does not affect the post's score.</p></li> 
              <li><p>2011-07-13: New heuristics <a href="http://meta.stackexchange.com/questions/27352/auto-detection-of-non-indented-code/98436#98436">blocking posts with no or improper code formatting</a> (only for users with &lt;= 50 reputation).</p></li> 
              <li><p>2011-07-08: <a href="http://meta.stackexchange.com/questions/53916/inline-editing-for-stack-exchange-sites">Inline post editing</a> is now enabled network wide to people with the editing privilege. </p></li> 
              <li><p>2011-07-07: It is no longer possible to award a bounty <a href="http://meta.stackexchange.com/questions/54998/i-just-awarded-myself-a-bounty/97802#97802">to your own answer</a>.</p></li> 
              <li><p>2011-07-07: 10k users can now see the Migrated tab on <a href="/tools?tab=migrated">/tools?tab=migrated</a>, which was previously a ♦ moderator tool.</p></li> 
              <li><p>2011-07-07: <a href="http://meta.stackexchange.com/badges/150/synonymizer"><em>Synonymizer</em> badge</a> added for <a href="http://meta.stackexchange.com/questions/97822/what-is-the-synonymizer-badge">First approved tag synonym</a>.</p></li> 
              <li><p>2011-07-07: Migrating to another site requires <a href="http://meta.stackexchange.com/questions/89242/require-more-than-simple-majority-of-close-votes-to-migrate/97700#97700">4 migration votes</a>, instead of a simple 3/5 majority as before. This change applies only to Stack Overflow.</p></li> 
              <li><p>2011-07-06: Close- and reopen- votes now <a href="http://meta.stackexchange.com/questions/56614/what-happened-here-voted-to-close-4-days-ago-but-no-close-votes-today/97594#97594">age away on a sliding window</a>: if four days elapse without a close or reopen vote, the votes begin aging away. Posts with fewer than 100 views do not expire votes.</p></li> 
              <li><p>2011-07-01: In comments between the post owner and only one other person, <a href="http://meta.stackexchange.com/questions/97098/eeeeek-what-happened-to-my-salutation">@lerting the post owner will be removed from the comment text</a> if it is at the beginning of the comment as it is not needed. (The post owner is always notified of comments on their post.) Moreover, @postowner (or an invalid @lert) does no longer block a subsequent @lert; see #8 in <a href="http://meta.stackexchange.com/questions/43019/how-do-comment-replies-work/43020#43020">How do comment @replies work?</a></p></li> 
             </ul>

             <h3>December 2013</h3> 
             <ul> 
              <li><p>2013-12-23: <a href="http://meta.stackexchange.com/questions/43004/how-do-i-audit-my-reputation/43005#43005">There is no "Trigger Reputation Recalc" button anymore.</a><br> Recalc is performed automatically only.<br> In case of reputation inconsistency that does not resolve itself, please file a <a href="http://meta.stackexchange.com/questions/ask?tags=bug,reputation">new bug report</a>.</p></li> 
              <li><p>2013-12-20: Personal data is <a href="http://meta.stackexchange.com/questions/213032/private-section-in-the-profile">now marked private</a> when viewing one's own profile.</p></li> 
              <li><p>2013-12-19: Posts <a href="http://meta.stackexchange.com/questions/184879/collapse-posts-when-flag-has-been-handled-in-10k-tools">now fade out</a> when flags have been handled in the 10K tools.</p></li> 
              <li><p>2013-12-04: <a href="http://blog.stackoverflow.com/2013/12/a-new-top-bar-for-stack-exchange/">New top bar rolling out on sites</a>.</p></li> 
              <li><p>2013-12-03: The 10K tool "post feedback" aka "anonymous and low rep user feedback" <a href="http://meta.stackexchange.com/a/209787/152859">is no more</a>. R.I.P</p></li> 
              <li><p>2013-12-03: <a href="http://meta.stackexchange.com/a/209777/165773">Direct linking to a filtered /review queue</a> is now available.</p></li> 
              <li><p>2013-12-03: New <a href="http://meta.stackexchange.com/questions/209775/extend-the-new-hot-questions-sidebar">Hot Questions sidebar</a> has been introduced in the homepage of each site, "recent tags" and "recent badges" lists <a href="http://meta.stackexchange.com/questions/209906/eeeek-hot-network-questions-killed-recent-tags-and-recent-badges">removed</a>.</p></li> 
             </ul> 
             <h3>November 2013</h3> 
             <ul> 
              <li><p>2013-11-27: <a href="http://meta.stackexchange.com/a/209171/21960">A user's remaining deletion votes are now displayed</a> on the "delete post" dialog.</p></li> 
              <li><p>2013-11-26: <a href="http://meta.stackexchange.com/a/190407/152859">"add comment" link is always visible</a>, even for users with less than 50 reputation points and anonymous visitors.</p></li> 
              <li><p>2013-11-14: <a href="http://meta.stackexchange.com/q/207420/165773">The new top bar (beta) is out on Meta Stack Overflow</a>.</p></li> 
              <li><p>2013-11-07 (or earlier): when calculating the Gravatar email hash for those who do not have a Gravatar account, in some cases <a href="http://meta.stackexchange.com/questions/204523/unexpectedly-changing-identicon/204540#204540">now a salt is used</a>.</p></li> 
             </ul> 
             <h3>October 2013</h3> 
             <ul> 
              <li><p>2013-10-08: Deleted answers that have had any spam or offensive flags raised against them will not show the answer body but the following text: "This answer was marked as spam or offensive and is therefore not shown - you can see the revision history for details". Where "the revision history" links to the answer revision history. The vote box (with score) and comments will still appear. (<a href="http://meta.stackexchange.com/a/199969">source</a>)</p></li> 
              <li><p>2013-10-07: In user history, Moderators can now see if a user is post-blocked, when, and if they've tried to post since being blocked. (<a href="http://meta.stackexchange.com/a/199758">source</a>)</p></li> 
             </ul> 
             <h3>September 2013</h3> 
             <ul> 
              <li><p>2013-09-20: Duplicates can now <a href="http://meta.stackexchange.com/questions/163628/new-stack-exchange-search-engine-ignores-automatic-possible-duplicate-blocks-i/197682#197682">be filtered for in search strings</a>.</p></li> 
              <li><p>2013-09-11: The dialog for flagging posts was redesigned in order to streamline the process. For example, the options for spam and offensive are now listed first. Also, duplicate votes are now separated from close votes.</p></li> 
              <li><p>2013-09-06: Any approved edit on a closed question <a href="http://meta.stackexchange.com/a/196078/165773">will push the post into the reopen queue</a>.</p></li> 
              <li><p>2013-09-05: New users (&lt; 10 rep) can now <a href="http://meta.stackexchange.com/a/195927/146318">post images on any site except for Stack Overflow, Server Fault, Super User and Ask Ubuntu</a>.</p></li> 
             </ul> 
             <h3>August 2013</h3> 
             <ul> 
              <li><p>2013-08-12?: It is possible to see <a href="http://meta.stackexchange.com/questions/185491/what-is-the-recently-deleted-questions-page-in-the-user-profile">one's own deleted recent answers and questions</a>. <code>http://stackoverflow.com/users/recently-deleted-answers/&lt;userID&gt;</code> and <code>http://stackoverflow.com/users/recently-deleted-questions/&lt;userID&gt;</code>. Linked from "answers" and "questions" tag in profile page.</p></li> 
              <li><p>2013-08-05: Closing a question will <a href="http://meta.stackexchange.com/questions/190947/is-it-time-to-be-rid-of-automatic-down-votes-on-certain-closed-questions/190954#190954">no longer incur an automatic Community downvote</a>.</p></li> 
              <li><p>2013-08-01: (estimated date) Custom comments in flags now <a href="http://meta.stackexchange.com/a/214270/152859">supports the same "mini-markdown" renderer used for comments</a>.</p></li> 
             </ul> 
             <h3>July 2013</h3> 
             <ul> 
              <li><p>2013-07-29: All flags that appear in a user's flagging summary that are not yet dismissed by a moderator have the marking "active" in it. It appears to mean "awaiting attention by a moderator". <a href="http://meta.stackexchange.com/questions/190738/what-does-active-mean-in-my-flagging-summary-page">Source</a>.</p></li> 
              <li><p>2013-07-26: As a counter-measure against robo-reviewing, review ban times <a href="http://meta.stackexchange.com/questions/174118/">now automatically increase</a> based on the amount of previous bans up to a maximum of 30 days.</p></li> 
              <li><p>2013-07-26: The reputation requirement to access the First Posts and Late Answers queue was bumped up to <a href="http://meta.stackexchange.com/questions/186373/propose-a-privilege-to-sit-in-the-500-reputation-mark">fill the empty 500 reputation privilege</a></p></li> 
              <li><p>2013-07-24: <a href="http://meta.stackexchange.com/q/190200">Alpha version of android app is released</a>.</p></li> 
              <li><p>2013-07-24: <a href="http://meta.stackexchange.com/a/190639">Very Low Quality flags automatically add the flagged post to the Low Quality review queue</a>. If more than one VLQ flag accumulates on the post before the review has ended, a number of "Looks Good" responses equivalent to the number of flags is required to dequeue it.</p></li> 
              <li><p>2013-07-19: The <a href="http://meta.stackexchange.com/a/189542">comment textarea now features the hint</a>: "Request clarification or suggest improvements." On Meta it is "Request clarification or add information."</p></li> 
              <li><p>2013-07-15: Close votes <a href="http://meta.stackexchange.com/questions/915/can-we-have-the-ability-to-rescind-a-close-vote-before-it-closes#comment582472_915">can now be retracted</a>.</p></li> 
              <li><p>2013-07-08: When an abandoned question is auto deleted, <a href="http://meta.stackexchange.com/a/237993/152859">there is a proper reason added to the post history record</a>.</p></li> 
              <li><p>2013-07-07: Comment flags are now <a href="http://meta.stackexchange.com/q/180377/166155">listed in full detail in a user's flag history.</a></p></li> 
              <li><p>2013-07-03: When a moderator accesses private information about a user (such as their e-mail address, full name and IP address), <a href="http://meta.stackexchange.com/q/187227">this action will be logged</a> in the user's history.</p></li> 
              <li><p>2013-07-03: Comments <a href="http://meta.stackexchange.com/questions/98036/moderators-should-be-able-to-undelete-comments">can now be undeleted by moderators</a>, unless they were deleted by the OP of the comment.</p></li> 
              <li><p>2013-07-03: <a href="http://meta.stackexchange.com/a/184309/165773">Close review history made available to 10K users and mods</a>. Later, other review histories were also made available (no official announcement found for that).</p></li> 
              <li><p>2013-07-02: <a href="http://meta.stackexchange.com/a/187008/152859">Flags to close a question are no longer shown in the moderator only queue</a>.</p></li> 
             </ul> 
             <h3>June 2013</h3> 
             <ul> 
              <li><p>2013-06-27: <a href="http://meta.stackexchange.com/questions/186345/">New design for privilege page</a> with privilege categorization and a progress bar to track progress to the next privilege. </p></li> 
              <li><p>2013-06-26: The question and answer ban messages <a href="http://meta.stackexchange.com/questions/183285/use-the-new-helpcenter-for-the-question-answer-ban-message/184182#184182">now link to the help center</a>, and the help center <a href="http://meta.stackoverflow.com/help/question-bans">pages</a> no longer link to <a href="http://meta.stackexchange.com/questions/86997/what-can-i-do-when-getting-sorry-we-are-no-longer-accepting-questions-answers/86998#86998">What can I do when getting “Sorry, we are no longer accepting questions/answers from this account”?</a></p></li> 
              <li><p>2013-06-26: <a href="http://meta.stackexchange.com/a/185890/165773">Flagging limits changes</a>: You start with 10 flags per day, you can end up with as many as 100 per day. You get an extra flag per day for every 2000 reputation points or 10 <em>net</em> helpful flags (<code>helpful-unhelpful</code>).</p></li> 
              <li><p>2013-06-26: <a href="http://meta.stackexchange.com/q/184154/165773">Closing changes: [on hold], unclear, too broad, opinion-based, off-topic reasons, bye-bye to Too Localized</a>, rolled out network-wide. See also <a href="http://blog.stackoverflow.com/2013/06/the-war-of-the-closes/">The War of the Closes</a></p></li> 
              <li><p>2013-06-19: Badges page is incorporated into the Help Center and <a href="http://meta.stackexchange.com/a/185013/187824">gets a new tabbed design</a>, cross network. For example, <a href="http://stackoverflow.com/help/badges">Stack Overflow badges page</a>.</p></li> 
              <li><p>2013-06-03: The <a href="http://meta.stackexchange.com/q/182812/212576">new Help Center</a> replaces FAQ pages on all sites.</p></li> 
              <li><p>2013-06-??: Tags are <a href="http://meta.stackexchange.com/a/234939">being auto suggested</a> while writing a new question or editing an existing question. (No official post at the time of implementation, basing the time on <a href="http://meta.stackexchange.com/questions/183620/dont-suggest-tags-that-a-question-has-already-been-tagged-with">this</a>)</p></li> 
             </ul> 
             <h3>May 2013</h3> 
             <ul> 
              <li><p>2013-05-31: <a href="http://meta.stackexchange.com/a/182572/165773">Collaborative effort lock</a> experimental feature introduced for questions where the primary value arises from the community's regular efforts to keep them updated via edits. </p></li> 
              <li><p>2013-05-28: Diamond moderators are now <a href="http://meta.stackexchange.com/q/158439/152859">allowed to comment on deleted or locked items</a>.</p></li> 
              <li><p>2013-05-22: Edit timestamp of posts <a href="http://meta.stackexchange.com/q/181361/152859">is now wholly clickable, including the "Edited" word itself</a>.</p></li> 
              <li><p>2013-05-17: Edits reverted during the grace period <a href="http://meta.stackexchange.com/a/180564/134300">will now remain in the post history</a> instead of vanishing.</p></li> 
              <li><p>2013-05-08: <a href="http://meta.stackexchange.com/a/179401/212576">Retag option from questions is removed.</a> Editing tags only will result in the same behavior as retag.</p></li> 
             </ul> 
             <h3>April 2013</h3> 
             <ul> 
              <li><p>2013-04-23: <a href="http://meta.stackexchange.com/a/177675">Changes to the treatment of deleted content</a>: Deleted questions will be visible to their authors, regardless of those authors' reputation; and delete-voting within two days of closing is disabled on questions scoring -2 or above, regardless of the voter's reputation.</p></li> 
              <li><p>2013-04-22: <a href="http://meta.stackexchange.com/a/177481">A new footer</a> is implemented on all sites. It links to a new <a href="http://stackexchange.com/sites?view=grid">grid view</a> of all Stack Exchange sites.</p></li> 
              <li><p>2013-04-02: Users are now shown a persistent (but dismissable) <a href="http://i.stack.imgur.com/hKsfX.png" rel="nofollow">banner</a> linking to <code>/about</code> on the site.</p></li> 
             </ul> 
             <h3>March 2013</h3> 
             <ul> 
              <li><p>2013-03-26: The <em>Linked</em> and <em>Related</em> lists on the right sidebar are now displayed with larger text and with the score for each question. </p></li> 
              <li><p>2013-03-26: The <em>Close</em> (or <em>Delete</em>) link <a href="http://meta.stackexchange.com/questions/172721/close-counter-overclutters-the-links-on-a-question">shows only the number of votes cast</a>. A tooltip (on hovering) displays the remaining required votes.</p></li> 
              <li><p>2013-03-22: If a closed question is edited by the asker within 5 days, <a href="http://meta.stackexchange.com/questions/157584/should-questions-be-added-to-the-reopen-queue-as-soon-as-theyre-edited-by-the-o#comment514253_157584">it is added to the reopen queue</a>.</p></li> 
              <li><p>2013-03-20: Users are now <a href="http://meta.stackexchange.com/a/172760/134300">blocked from posting comments consisting only of "what have you tried"</a>.</p></li> 
              <li><p>2013-03-19: A "delete" link <a href="http://meta.stackexchange.com/a/172428/152859">has been added</a> after "edit" for comments on mobile theme.</p></li> 
              <li><p>2013-03-14: A question (or answer) with Votes to Close (or Delete) now shows the number of votes already cast and the number of votes needed to close (or delete) (e.g., <em>(2/5)</em>)</p></li> 
             </ul> 
             <h3>February 2013</h3> 
             <ul> 
              <li><p>2013-02-21: New image upload is being tested on Meta, adding <a href="http://meta.stackexchange.com/questions/80201/upload-images-by-drag-and-drop">support for paste and drag-and-drop</a>.</p></li> 
              <li><p>2013-02-20: [About] in comments <a href="http://meta.stackexchange.com/q/163986/152859">is now a magic link</a>, translated automatically as a link to the site's about page. (e.g. <a href="http://stackoverflow.com/about">http://stackoverflow.com/about</a> for Stack Overflow)</p></li> 
              <li><p>2013-02-19: When a comment thread contains a comment by a user whose name starts with a non-ASCII letter (e.g. Я) and you write new comment, <a href="http://meta.stackexchange.com/a/168155/152859">just typing @ will bring the tab completer up with that name</a>.</p></li> 
              <li><p>2013-02-18: The revision history of a question that was closed as a duplicate is now <a href="http://meta.stackexchange.com/a/167985/152859">showing the chosen originals</a> as part of the "Closed as duplicate" notice.</p></li> 
              <li><p>2013-02-11: If a visitor is not logged in or doesn't have enough reputation to vote, the vote system redirects the vote to the anonymous feedback system, and shows a <a href="http://i.stack.imgur.com/0u1a7.png" rel="nofollow">small blue dialog</a>.</p></li> 
              <li><p>2013-02-06: The auto inserted text for questions closed as duplicates has been changed to <em>"This question already has an answer here:"</em> followed by link to the other question and the number of answers it has. The text is no longer edited into the original question but rather displayed externally. While the question is still open, <a href="http://meta.stackexchange.com/a/167980/152859">only the OP sees the message</a> and it becomes visible to everyone once the question gets closed. Questions closed as duplicate prior to this change still have the old <em>"Possible Duplicate:"</em> text in them.</p></li> 
              <li><p>2013-02-06: The close/flag/bounty dialogs <a href="http://meta.stackexchange.com/questions/166392/can-we-put-a-border-back-on-the-flag-dialog">lose their border</a>, and are made <a href="http://meta.stackexchange.com/questions/74976/the-new-flagging-dialog-gets-in-the-way">draggable</a></p></li> 
              <li><p>2013-02-01: The "close as duplicate" process <a href="http://meta.stackexchange.com/q/165933/134300">has been updated</a>: the interface to search for duplicate questions has been improved, and users may no longer vote to close questions as duplicates of questions that do not have an upvoted or accepted answer</p></li> 
             </ul> 
             <h3>January 2013</h3> 
             <ul> 
              <li><p>2013-01-29: Automatic <a href="http://meta.stackexchange.com/tags/review-suspension/info" title="what's this">review suspension</a> has been <a href="http://meta.stackexchange.com/a/165417/165773" title="as explained here">introduced</a>. We're kicking blatant <a href="http://meta.stackexchange.com/tags/review-audits/info" title="review audits are used to automatically detect abuse">abusers</a> out of the queues automatically now. Included are <a href="http://meta.stackexchange.com/questions/159544/why-no-honeypot-suggested-edits">often-hilarious "honeypot" fake suggested edits</a>.</p></li> 
              <li><p>2013-01-22: Accept rates <a href="http://meta.stackexchange.com/a/164654/134300">will no longer be shown</a>. Like flag weight, they will continue to be calculated for backend use.</p></li> 
              <li><p>2013-01-22: A question closed as a duplicate <a href="http://meta.stackexchange.com/questions/90672/can-we-make-the-title-of-duplicate-questions-say-duplicate/164651#164651">now gets</a> a "&nbsp;[duplicate]" appended to its title, rather than just "&nbsp;[closed]". It also shows "marked as duplicate" instead of "closed as…" in the post notice.</p></li> 
              <li><p>2013-01-10: First version of <a href="http://meta.stackexchange.com/questions/163254/were-rolling-out-a-new-quick-start-guide-to-help-new-users-learn-the-basics">the new 'About' page rolled out</a> to act as Quick Start guide for new users. New Bronze "Informed" badge for reading the entire page. </p></li> 
              <li><p>2013-01-10: If <em>you</em> voted to close a post, and it hasn't been edited since the time it was closed,<br> <a href="http://meta.stackexchange.com/a/163173/165773">then you won't see that post in the Reopen Queue</a>.</p></li> 
              <li><p>2013-01-10: An <a href="http://meta.stackexchange.com/questions/149436/disable-gravatar-on-stack-overflow/161083#161083">alternative to Gravatar</a> has been rolled out network-wide.</p></li> 
              <li><p>2013-01-04: A new elasticsearch-based search engine <a href="http://meta.stackexchange.com/questions/160100/a-new-search-engine-for-stack-exchange">has been rolled out across the network</a>. New features include multi-term searches defaulting to AND rather than OR, word stemming, excluded terms <code>-term</code>, <code>infavorites:</code> coming back, ranges for <code>answers:</code> and <code>created:</code> and <code>score:</code>, and more.</p></li> 
             </ul>

             <h3>December 2012</h3> 
             <ul> 
              <li><p>2012-12-14: <a href="http://meta.stackexchange.com/q/158762/178438">CSS of Stack Overflow and Meta Stack Overflow</a> has been updated, mainly "thinning down" text.</p></li> 
              <li><p>2012-12-14: <a href="http://meta.stackexchange.com/a/158761/165773">"No Action Needed" button</a> has been introduced in First Posts and Late Answers review queues.</p></li> 
              <li><p>2012-12-13: <a href="http://meta.stackexchange.com/a/158505/165773">Suspending reviewing privileges</a> has been introduced for those who fail multiple <a href="http://meta.stackoverflow.com/tags/review-audits/info" title="what's this">review audits</a> in a short time.</p></li> 
              <li><p>2012-12-03: When an abandoned question is auto deleted, <a href="http://meta.stackexchange.com/a/237993/152859">a post history record is being created and attributed to the Community user</a>.</p></li> 
             </ul> 
             <h3>November 2012</h3> 
             <ul> 
              <li>2012-11-30: <a href="http://meta.stackexchange.com/a/152596/165773"><em>Vote to close -&gt; offtopic</em> from Stack Overflow</a> adds <a href="http://sharepoint.stackexchange.com/faq">SharePoint</a>; removes <a href="http://serverfault.com/faq">Server Fault</a>.</li> 
             </ul> 
             <h3>October 2012</h3> 
             <ul> 
              <li><p>2012-10-19: <a href="http://meta.stackexchange.com/a/147981/165773">Review actions limits have been changed</a>: after 20 reviews in a given queue, you're done for the day - if you want to continue reviewing, you can switch to a different queue. Limit is doubled for queues with large backlogs.</p></li> 
              <li><p>2012-10-18: <a href="http://meta.stackexchange.com/q/151890/133693">Migration for questions older than 60 days is now not possible, even by Moderators</a></p></li> 
              <li><p>2012-10-17: Reopen Votes review queue <a href="http://meta.stackexchange.com/a/151618/165773">has been introduced</a> to vote whether or not to reopen closed questions.</p></li> 
              <li><p>2012-10-16: The number of reviews required to accept/reject suggested edits <a href="http://meta.stackexchange.com/a/151586/152859">has been increased</a> to two network wide (versus one before) and three on Stack Overflow (versus two before). Moderators can still cast the final vote like before.</p></li> 
             </ul> 
             <ul> 
              <li>2012-10-01 (likely earlier): <a href="http://i.stack.imgur.com/nxQPB.png" rel="nofollow">suspension reason</a> "This account is temporarily suspended for asking programming questions on MSO" introduced.</li> 
             </ul> 
             <h3>September 2012</h3> 
             <ul> 
              <li><p>2012-09-22: <a href="http://meta.stackexchange.com/a/141145/165773">Voting allowed</a> (and encouraged) in First Posts and Late Answers review queues.</p></li> 
              <li><p>2012-09-21: <a href="http://meta.stackexchange.com/questions/147912/new-community-review-badges">New badges added</a> for completing community review tasks. User can earn <a href="http://stackoverflow.com/badges/2278/custodian">Custodian</a> (1 review), <a href="http://stackoverflow.com/badges/1478/reviewer">Reviewer</a> (250 reviews) and <a href="http://stackoverflow.com/badges/2279/steward">Steward</a> (1000 reviews) once for each review queue (Close Posts, First Posts, Late Answers, Suggested Edits and Low Quality Posts). The previous Reviewer badge (for reviewing posts in the old system) will no longer be awarded.</p></li> 
              <li><p>2012-09-08: You must be a <a href="http://meta.programmers.stackexchange.com/a/4001/31260">registered user to ask questions</a> on Programmers Stack Exchange. Unregistered users may still answer questions however.</p></li> 
             </ul> 
             <h3>August 2012</h3> 
             <ul> 
              <li><p>2012-08-15: Canned <a href="http://meta.stackexchange.com/a/143640">Comment Feedback Dialog</a> for <a href="http://meta.stackexchange.com/q/117963">Community Review</a> page.</p></li> 
              <li><p>2012-08-13: <a href="http://meta.stackexchange.com/a/143380/165773">Review audits</a> are introduced: known cases thrown into review queue as "gotchas" to keep people from just nailing "Looks Good" or "Close" or whatever without reading the posts.</p></li> 
             </ul> 
             <h3>July 2012</h3> 
             <ul> 
              <li><p>2012-07-13: The <a href="http://stackoverflow.com/help">Help Center</a> has been added. It contains pages for many common issues, offering suggestions and a form to email the team as needed. This is the new target of the "contact us" link in the footer, which was previously an email link.</p></li> 
              <li><p>2012-07-12: The new <a href="http://meta.stackexchange.com/questions/139536/new-feature-community-review-tasks-now-in-beta">Community Review Tasks system is now in beta on the trilogy</a>.</p></li> 
              <li><p>2012-07-06: The edit link <a href="http://meta.stackexchange.com/a/138944/134300">is now disabled instead of hidden for registered users who cannot edit the post</a>. The tooltip will display a reason such as "Post is locked".</p></li> 
              <li><p>2012-07-02: "question eligible for bounty in X" message <a href="http://meta.stackexchange.com/a/138428/152859">will be hidden <em>for the question owner</em> only if the question has an accepted answer</a>. (Before this change it was hidden when question got an answer)</p></li> 
             </ul> 
             <h3>June 2012</h3> 
             <ul> 
              <li><p>2012-06-08: New badges <a href="http://stackoverflow.com/badges/1973/caucus">Caucus</a> and <a href="http://stackoverflow.com/badges/1974/constituent">Constituent</a>, for browsing an election page and for voting in an election; <a href="http://blog.stackoverflow.com/2012/06/election-participation-badges/">announced on the blog</a>.</p></li> 
              <li><p>2012-06-05: <a href="http://meta.stackexchange.com/a/134642/157556"><em>Vote to close -&gt; offtopic</em> from Stack Overflow</a> adds <a href="http://tex.stackexchange.com/faq">TeX</a>, <a href="http://dba.stackexchange.com/faq">DBA</a>; removes <a href="http://programmers.stackexchange.com/faq">Programmers</a> and <a href="http://webmasters.stackexchange.com/faq">Webmasters</a>.</p></li> 
             </ul> 
             <h3>May 2012</h3> 
             <ul> 
              <li><p>2012-05-21: <a href="http://blog.stackoverflow.com/2012/05/revamped-notifications/">Notifications tab</a> in the multicollider (next to the inbox).</p></li> 
              <li><p>2012-05-18: You can <a href="http://meta.stackexchange.com/questions/132886/what-is-this-answer-your-own-question-jazz">answer your own question</a> at the time of posting by ticking the “Answer your own question” checkbox below the “Post Your Question” button.</p></li> 
              <li><p>2012-05-10: When there is a community moderator election <a href="http://meta.stackexchange.com/a/132189/163868">it is noted in that site's newsletter</a>.</p></li> 
              <li><p>2012-05-05: Keyboard shortcuts (created with <code>&lt;kbd&gt;</code>) on all sites <a href="http://meta.stackexchange.com/a/131447/159624">are now rendered</a> like they've always been on <a href="http://apple.stackexchange.com">Ask Different</a>, with a less boxy layout.</p></li> 
             </ul> 
             <h3>Apr 2012</h3> 
             <ul> 
              <li><p>2012-04-30: Suggested edits that change the post's title now have <a href="http://meta.stackexchange.com/questions/79342/title-edit-plus-minor-body-edit-return-too-short-error/130741#130741">no minimum on the body edit</a>.</p></li> 
              <li><p>2012-04-26: Stack Exchange is experimenting with <a href="http://meta.stackexchange.com/questions/130398/does-stack-exchange-crawl-web-sites">automatically following links</a> and might post comments and add them to a list on the <code>/review</code> tab when they are dead.</p></li> 
              <li><p>2012-04-26: <a href="http://meta.stackexchange.com/a/130382/165773">New votes tab on user profile</a>, clearly you can only see your votes.<br> You can see deletions/undeletions/closures and reopens as well as up/down votes (<a href="http://meta.stackexchange.com/a/130380/165773">source</a>).</p></li>

              <li><p>2012-03-15: When anonymous users visit a question that is closed as duplicate and has no answer, they are <a href="http://meta.stackexchange.com/questions/125220/automatically-redirect-anonymous-user-from-unanswered-duplicate-question-to-corr/125807#125807">automatically redirected to the duplicate</a>.</p></li> 
              <li><p>2012-03-15: <a href="http://meta.stackexchange.com/questions/125677/new-feature-real-time-updates-to-questions-answers-and-inbox">'Live updates' implemented on Stack Exchange sites</a>; due to heavy activity, the feature is limited to tags view on Stack Overflow.</p></li>

              <li><p>2012-01-30: Suggested edits can now be <a href="http://meta.stackexchange.com/questions/83721/do-we-need-a-reject-and-improve-button/109769#109769">marked as unhelpful</a> when selecting the "Improve" option, causing the Community user to decline the suggested edit with no reputation bonus to the suggester.</p></li>

Users having flags now see a "helpful flags" count on their own profile linking to the <a href="http://meta.stackexchange.com/tags/flagging-summary/info">flag audit page</a>, which now contains a breakdown of flag outcomes by flag type (moderator attention, spam, offensive, comment). The <a href="http://meta.stackoverflow.com/badges/140/deputy">Deputy</a> and <a href="http://meta.stackoverflow.com/badges/160/marshal">Marshal</a> badges were modified to require a number of helpful flags (80 and 500, respectively) instead of a specific flag weight.</p></li>

              <li><p>2012-01-16: The populist badge <a href="http://meta.stackexchange.com/a/119221/134300">will no longer be awarded for self-answers</a>.               <li><p>2012-01-10: The Announcer, Booster and Publicist badges can now be <a href="http://meta.stackexchange.com/questions/118484/booster-badge-should-be-golden-or-easier-to-achieve/118509#118509">earned multiple times and they can also be earned for the same question</a>. The time limit has also been removed. Sharing direct links to answers now <a href="http://chat.meta.stackoverflow.com/transcript/message/816147#816147">also counts</a> towards these badges.</p></li> 
              <li><p>2012-01-03: Users asking questions with auto-caught bad titles <a href="http://meta.stackexchange.com/a/117788">will now be offered</a> a link to <a href="http://meta.stackexchange.com/a/10648">Mark Harrison's post on writing good titles</a>.</p></li>

              <li><p>2014-04-16: Meta Stack Overflow and Meta Stack Exchange have <a href="http://meta.stackexchange.com/q/228888">officially split off from each other</a>.</p></li> 
              <li><p>2014-04-14: The last activity information on the front page <a href="http://meta.stackexchange.com/questions/145039/can-we-add-asked-answered-edited-text-to-each-question-featured-on-the-f/229665#229665">now includes verbs</a>: asked, answered, or modified (that covers edits, bounties, and the like.)</p></li> 
              <li><p>2014-04-09: <a href="http://meta.stackexchange.com/questions/227290/stop-using-community-wiki-as-a-reputation-denial-mechanism/228940#228940">Former criteria for the automatic conversion of community wiki have been removed</a>. Instead, flags will be cast in select situations for moderator attention regarding multiple edits to a post or multiple answers to a post.</p></li> 
              <li><p>2014-04-09: Moderators can now <a href="http://meta.stackexchange.com/a/228927">provide a reason when manually review-banning users</a>. In addition, the message on automatic bans triggered by failed audits includes a link to at least one such failed audit.</p></li> 
              <li><p>2014-04-08: <a href="http://meta.stackexchange.com/a/228805">Auto-protection for questions that get more than N answers from new users in a 24-hour period</a>. <code>N</code> is: 5 by default, 20 for Code Golf, 3 for Programmers, Workplace, ELL. Users with privileges to protect (15K) can now unprotect any protected question.</p></li> 
              <li><p>2014-04-05: When a post is deleted by six "Recommend Deletion" votes from within the review queue, <a href="http://meta.stackexchange.com/a/184568">"Post Deleted From Review" message will be added to the post history</a>. In addition, when the post author undelete such a post, <a href="http://meta.stackexchange.com/a/228479">"disputed low quality review" auto flag is raised</a>.</p></li> 
              <li><p>2014-04-??: <a href="http://meta.stackexchange.com/q/214563">Chat now has the new top bar</a>, as do <a href="http://meta.stackexchange.com/q/210357">Area&nbsp;51 and the StackExchange.com hub</a>.</p></li> 
              <li><p>2014-04-??: <a href="http://meta.stackexchange.com/a/230421">Tag popup contains amount of questions having that tag</a>.</p></li> 
             </ul> 
             <h3>March 2014</h3> 
             <ul> 
              <li><p>2014-03-19: <a href="http://meta.stackexchange.com/questions/215450/ssl-breaks-tex-rendering/225923#225923">MathJax now works over HTTPS</a>.</p></li> 
              <li><p>2014-03-17: Moderators can now <a href="http://meta.stackexchange.com/a/225779">dismiss flags on the same post individually</a>.</p></li> 
              <li><p>2014-03-11: Moderator Newsletter updates are now pushed to the Top Bar Moderator Inbox.</p></li> 
              <li><p>2014-03-11: Top Bar Moderator Inbox now includes snippets from moderator messages in the alerts.</p></li> 
             </ul>

              <li><p>2014-02-28: <a href="http://meta.stackexchange.com/a/223601">UI for moderators to reply to helpful flags</a>.</p></li> 
              <li><p>2014-02-27: The number of questions in the close review queue <a href="http://meta.stackexchange.com/a/223349">is now fuzzy</a>.</p></li> 
              <li><p>2014-02-25: The Top Bar now uses <a href="http://meta.stackexchange.com/questions/212372/top-bar-should-be-consistent-on-all-sites-but-its-not/223003#223003">universal colors network-wide</a> when depicting flags (blue) and suggested edits (beige).</p></li> 
              <li><p>2014-02-25: Hot questions list at sidebar <a href="http://meta.stackexchange.com/questions/222991/ease-up-on-the-hot-network-question-randomness/222993#comment726179_222993">is now randomized</a>.</p></li> 
              <li><p>2014-02-25: The order and choice of sites on the top-left site switcher <a href="http://meta.stackexchange.com/q/215928">can now be customized</a>.</p></li> 
              <li><p>2014-02-13: Comments on election nominations can no longer be flagged. There will be extra monitoring to ensure this doesn't result in abusive comments.</p></li> 
              <li><p>2014-02-10: <a href="http://meta.stackexchange.com/q/220264">iOS app Alpha version released</a>.</p></li> 
              <li><p>2014-02-03: Improving a suggested edit is now considered more substantive than an approved suggested edit (<a href="http://meta.stackexchange.com/a/218484">details</a>).</p></li> 
             </ul>

              <li><p>2014-01-27: <a href="http://meta.stackexchange.com/q/215992">The Stack Exchange Android app is in public beta</a>.</p></li> 
              <li><p>2014-01-24: The mobile site now uses a mini version of the <a href="http://blog.stackoverflow.com/2013/12/a-new-top-bar-for-stack-exchange/">improved topbar</a>.</p></li> 
              <li><p>2014-01-15: <a href="http://meta.stackexchange.com/a/216285">Migration notice refers to question revision history at source site</a>.</p></li> 
              <li><p>2014-01-??: Moderators can now see who validated a chat offensive flag in the flag dialog.</p></li>

             <h3>December 2015</h3> 
             <ul> 
              <li><p>2015-12-23: Question timeline (not moderator timeline) got a new design and features, e.g. <a href="http://meta.stackexchange.com/a/271797/152859">showing who voted to close or reopen</a>, and more.</p></li> 
              <li><p>2015-12-09: Users with 10k reputation can now <a href="http://meta.stackexchange.com/a/270562/152859">use the <code>deleted:1</code> advanced search operator to search their own deleted posts</a>.</p></li> 
             </ul> 
             <h3>November 2015</h3>

              <li>2015-11-09: Comments auto-added from review queues now <a href="//meta.stackoverflow.com/a/309704" rel="nofollow">indicate as such</a>.</li> 
             </ul> 
             <h3>October 2015</h3> 
             <ul> 
              <li><p>2015-10-23: It is now possible to <a href="http://meta.stackexchange.com/a/268369/168244">filter your flag history by type</a>.</p></li> 
              <li><p>2015-10-20: <a href="http://meta.stackexchange.com/a/268194/165773">More consistent “Flagged Posts” page</a>: "The stats presented should now be consistent across all flag types, including waiting for review, disputed, declined, aged away, etc."</p></li> 
              <li><p>2015-10-13: Rejected migrations now have an additional notice: "Votes, comments, and answers are locked due to the question being closed here, but it may be eligible for editing and reopening on the site where it originated." <a href="http://meta.stackexchange.com/q/199349">Example</a></p></li> 
              <li><p>2015-10-10: New Help Center page for automatic deletion, linked from deletion banners: "This question was automatically deleted. <a href="http://meta.stackexchange.com/help/roomba">Please see the help center for more information.</a>"</p></li> 
              <li><p>2015-10-08: Dupehammer got a new design, and now applies to <a href="http://meta.stackexchange.com/a/268281">current tags rather than original tags, but is disabled if you edited the tags</a>.</p></li> 
              <li><p>2015-10-05: Automatic deletions by Community are now <a href="http://meta.stackexchange.com/a/267439">annotated</a> with an indication of reason: one of RemoveMigrationStubs, RemoveDeadQuestions, RemoveAbandonedQuestions, and RemoveAbandonedClosed. <a href="http://meta.stackexchange.com/a/92006">Explanation of reasons</a>.</p></li> 
             </ul> 
             <h3>September 2015</h3> 
             <ul> 
              <li><p>2015-09-29: Bar for reputation for late answers to enter the review queue <a href="http://meta.stackexchange.com/a/267080/165773">raised from 10 to 50</a>.</p></li> 
              <li><p>2015-09-07: <a href="http://meta.stackexchange.com/a/253669/168244">New user privilege at 25K reputation</a>: <a href="http://meta.stackexchange.com/help/privileges/site-analytics">Access to Site Analytics</a></p></li> 
              <li><p>2015-09-02: Tag badges <a href="http://meta.stackexchange.com/q/254052/255554">can now be tracked</a> by users that have not acquired all privileges.</p></li> 
             </ul> 
             <h3>July 2015</h3> 
             <ul> 
              <li>2015-07-22: <a href="http://meta.stackexchange.com/a/261182/152859">Image uploader automatically wraps images in a link to themselves</a>.</li> 
             </ul> 
             <h3>June 2015</h3> 
             <ul> 
              <li>2015-06-16: Checking the "community wiki" box when posting an answer now <a href="http://i.stack.imgur.com/s0HNX.png" rel="nofollow">displays a confirmation dialog</a>.</li> 
             </ul> 
             <h3>May 2015</h3> 
             <ul> 
              <li>2015-05-05: "Inappropriate" flag option for posts renamed again, now it is <a href="http://chat.meta.stackexchange.com/transcript/89?m=3399658#3399658">"rude or abusive"</a>.</li> 
             </ul> 
             <h3>April 2015</h3> 
             <ul> 
              <li><p>2015-04-22: <a href="http://meta.stackoverflow.com/a/290915/">Flag dialog has been reworded</a> for both questions and answers. Closing categories are now stated on the primary screen; "offensive" renamed as "inappropriate".</p></li> 
              <li><p>2015-04-20: Questions deleted as spam/inappropriate now have the body and title hidden from view (<a href="http://i.stack.imgur.com/vmDZ2.png" rel="nofollow">example</a>), as was <a href="http://meta.stackexchange.com/a/199969/">previously done for answers</a>. For users under 10K, the "not found" page <a href="http://meta.stackexchange.com/q/217529/">no longer contains suggestions for search</a>. Also, there is no slug in the URL for such posts. </p></li> 
              <li><p>2015-04-15: New profile pages <a href="http://meta.stackexchange.com/q/253450">have been launched on most Stack Exchange sites</a>. These feature separate "Profile" and "Activity" sub-pages. &nbsp; See, also, the blog entry: <a href="http://blog.stackoverflow.com/2015/04/two-new-user-pages-one-new-stat-this-ones-big/"><em>Two new user pages. One new stat. This one’s big.</em></a>.</p></li> 
              <li>2015-04-08: <a href="http://meta.stackexchange.com/a/252991/165773">Moderators able to manually ban users from suggesting edits</a>.</li> 
             </ul> 
             <h3>March 2015</h3> 
             <ul> 
              <li>2015-03-23: <a href="http://meta.stackexchange.com/a/251876/">Criteria for making an entry in post revision history have changed</a>. Question edit grace period ends once an answer has been posted, and any post's grace period ends when someone other than the poster adds a comment. Rollbacks and edits of rollbacks now always create entries in the revision history.</li> 
              <li>2015-03-20: <a href="http://meta.stackoverflow.com/a/288387/839601"><em>All</em> questions flagged Very Low Quality now immediately enter Triage</a> instead of Low Quality review queue at Stack Overflow. And while these questions are being triaged, they're <em>not</em> shown on the home page.</li> 
              <li>2015-03-09: <a href="http://meta.stackexchange.com/a/250930">Clarify to the OP that their question is not yet closed and the duplicates are only suggestions</a>. Asker is presented with options: "That solved my problem! / No, my question is different. I will edit to explain how". The first option causes the <em>Community</em> user to immediately close the question as a duplicate; the second option presents the editing screen. (<a href="http://meta.stackexchange.com/questions/250981/new-ui-encourages-askers-to-confirm-or-dispute-duplicate-votes">Announcement</a>)</li> 
              <li><p>2015-03-05: <a href="http://stackoverflow.com/review/helper">The Help &amp; Improvement</a> review queue <a href="http://meta.stackoverflow.com/q/287466/447356">introduced for testing at Stack Overflow</a>.</p></li> 
              <li><p>2015-03-02: Modernized <a href="http://meta.stackexchange.com/q/250223/165773">close vote</a> and <a href="http://meta.stackexchange.com/a/250496/165773">close flag</a> aging logic. Aging votes and flags starts after 14 days, regardless of view count. Allowed re-casting votes and flags that've aged away after 14 days. <a href="http://meta.stackoverflow.com/questions/278380/help-us-test-question-triage">Triage review</a> (currently Stack Overflow only) instantly disputes flags if the consensus is not "unsalvageable". This would go into effect <a href="http://meta.stackexchange.com/questions/250223/age-close-votes-after-14-days-regardless-of-views-allowing-recasting#comment816968_250223" title="'Not going to try to re-write history...'">for current and future votes</a>.</p></li> 
             </ul> 
             <h3>February 2015</h3> 
             <ul> 
              <li><p>2015-02-23: New “User Activity” page <a href="http://meta.stackexchange.com/q/249951">has been launched on MSE</a>, seeking feedback from the community before applying it network wide.</p></li> 
              <li><p>2015-02-13: <a href="http://meta.stackexchange.com/q/248511/">Review page on main sites now links to reviews on meta site</a>.</p></li> 
              <li><p>2015-02-07: Questions that are too old to migrate (60+ days) are now <a href="http://meta.stackexchange.com/q/247112">explicitly stated as being too old when attempting suggest migration through flagging</a>.</p></li> 
              <li><p>2015-02-03: When opening the flagging box, if a flag was cast previously, now the system accurately identifies which kind of flag the user previously cast.</p></li> 
             </ul> 
             <h3>January 2015</h3> 
             <ul> 
              <li><p>2015-01-30: <a href="http://meta.stackexchange.com/a/239597/152859">Users can't vote to delete posts that are the target of duplicate(s)</a>. If a user (moderator or any other user) votes to delete a question that has duplicate questions linked to it, their vote won't go through and they would see a warning about why.</p></li> 
              <li><p>2015-01-28: <a href="http://meta.stackoverflow.com/a/270969/839601">ML / Prediction data made available</a> to users – <a href="http://stackoverflow.com/users/prediction-data">a page</a> that lets you see what our algorithms think, download the data, and disable the predictions if so desire.</p></li> 
              <li><p>2015-01-28: <a href="http://meta.stackexchange.com/a/247658/165773">Very Low Quality and Not An Answer flags do not enter the moderator queue for 15 minutes after they're raised.</a> This applies network-wide, except on Stack Overflow, Mathematics, TeX, Salesforce and Stack Overflow em Português (where the delay is a full hour) and meta sites (where they enter the queue immediately).</p></li> 
              <li><p>2015-01-26: When logged in to private beta, user can <a href="http://meta.stackexchange.com/a/247610/152859">now choose between real name and display name when inviting a friend to join</a>.</p></li>

              <li><p>2011-06-23: The auto-award period of bounties has been <a href="http://meta.stackexchange.com/questions/68050/a-grace-period-of-one-day-to-award-the-bounty-after-expiration-without-remaining/74037#74037">extended 24 hours past the true end of the bounty</a>.</p></li>

              <li><p>2011-06-15: The “subjective and argumentative” close reason has been replaced by <a href="http://meta.stackexchange.com/questions/89417/make-not-constructive-a-close-reason-across-the-stack-exchange-network/95338#95338">“not constructive”</a>.</p></li> 
              <li><p>2011-06-14: <a href="http://blog.stackoverflow.com/2011/02/suggested-edits-and-edit-review/">Suggested edits</a> are now <a href="http://meta.stackexchange.com/questions/92614/unable-to-suggest-edits-on-a-closed-question/95019#95019">enabled for closed questions</a>.</p></li> 
              <li><p>2011-06-10: When adding a new comment to a post with many comments between two users in a short time, a link is provided to <a href="http://meta.stackexchange.com/questions/95937/why-must-we-avoid-discussions-in-comments">create a chat room to continue the conversation</a>.</p></li>

People who suggest edits <a href="http://meta.stackexchange.com/questions/77505/edit-a-question-from-other-user-and-get-2-rep/77508#77508">can earn</a> up to 1000 reputation, 2 for each approved edit.</p></li> 
              <li><p>2011-01-27: Area 51 <a href="http://blog.stackoverflow.com/2011/01/area-51-gets-organized-lets-talk/">gets</a> an updated design, proposal categories, and a <a href="http://discuss.area51.stackexchange.com/">discussion zone</a>. A51 posts are now oneboxed in chat.</p></li> 
              <li><p>2011-01-27: Stack Exchange now uses <a href="http://meta.stackexchange.com/questions/76621/new-stack-exchange-search-try-it-out">Lucene instead of SQL Fulltext</a> for search, related questions, and the 'similar questions' in Ask a Question. Option <code>infavorites</code> is no longer supported, some search keys like <code>intitle</code> have changed names, and a new option <code>body</code> is introduced. See <a href="http://meta.stackoverflow.com/search">the help</a> on the <code>/search</code> pages.</p></li> 
              <li><p>2011-01-21: The three-character requirement for <a href="http://meta.stackexchange.com/questions/43019/how-do-comment-replies-work/">comment @replies</a> has been relaxed <a href="http://meta.stackexchange.com/questions/61960/allow-for-comment-reply-to-usernames-with-less-than-three-characters-in-first-word/75809#75809">for the first word</a>.               <li><p>2011-01-18: Sort option "oldest" <a href="http://meta.stackexchange.com/questions/71561/bring-back-the-oldest-sort-order-for-answers/75301#75301">is back</a>, replacing "newest."</p></li>

Subscribing to RSS is not tracked.</p></li>

Its "flag for moderator attention" option now provides several default reasons, <a href="http://meta.stackexchange.com/questions/74974/please-remove-the-unneeded-options-when-flagging-your-own-posts">where appropriate</a>: "not an answer", "low quality" and "should be CW". It allows for <a href="http://meta.stackexchange.com/questions/74252/allow-longer-comments-in-moderator-flags/74306#74306">longer messages</a> to a moderator. The daily maximum number of flags has been increased. And if a particular user keeps moderator flagging for reasons that are considered invalid, their flag weight decreases. And for those users who continually flag reliably, their flag weight increases.</p></li> 
              <li><p>2011-01-11: <a href="http://blog.stackoverflow.com/2011/01/twitter-question-feeds-for-stack-exchange">Twitter question feeds</a> to publish the most interesting questions on each Stack Exchange site (but <em>not</em> the SOFU trilogy) every three hours. Like: <a href="http://twitter.com/StackProgrammer" rel="nofollow">twitter.com/StackProgrammer</a>.</p></li>

In all other cases, voting or flagging is required.</p></li> 
              <li><p>2011-01-08: The "flags" tab in 10k tools was improved; it now shows comment flags in addition to spam/offensive flags, and lets users display the entire post and add their own flag directly from the flags tab. It also preserves the other mod tool tabs.</p></li>

From this you can subscribe to email notifications, add the tag to your RSS reader and view the tag wiki excerpt. Where no tag wiki is available an invitation to create the tag wiki appears. The pop-up also includes links to the FAQ, Info &amp; Top Users pages for the tag.</p></li>

              <li><p>2011-01-06: New "faq" tab added to the <code>/questions</code> page, <a href="http://meta.stackexchange.com/questions/74094/has-the-hot-tab-been-renamed-to-faq">displacing the "hot" tab</a> (but not removing "hot" functionality).</p></li> 
              <li><p>2011-01-06: Comments now support <a href="http://meta.stackexchange.com/questions/63366/would-it-be-possible-to-fix-links-containing-parentheses-in-comments">inserting links containing parentheses</a> e.g. <code>[foo](http://msdn.microsoft.com/en-us/library/ms535874(VS.85).aspx)</code></p></li> 
              <li><p>2011-01-03: Page footers now indicate when a <img src="http://i.stack.imgur.com/oZyvE.png" alt="new"> post has been made to the SO blog.</p></li>

              <li><p>2010-09-06: <a href="http://meta.stackexchange.com/questions/57594/allow-prepopulating-the-tags-field-when-linking-to-the-ask-question-page">The title and tags field of the Ask Question page can be pre-populated</a> by specifying the <code>title</code> and/or <code>tags</code> query parameters, like <a href="http://meta.stackexchange.com/questions/ask?title=Some+title&amp;tags=tag1+tag2"><code>/questions/ask?title=Some+title&amp;tags=tag1+tag2</code></a></p></li> 
              <li><p>2010-09-01: New badge, <a href="http://stackoverflow.com/badges/254/tag-editor">Tag Editor</a> (for first Tag Wiki edit) introduced.</p></li> 
             </ul> 
             <h3>Aug 2010</h3> 
             <ul> 
              <li><p>2010-08-24: Support for <a href="http://meta.stackexchange.com/questions/4773/tweet-this-button/62090#62090">shorter URLs</a>, like <a href="http://stackoverflow.com/q/1732348">stackoverflow.com/q/1732348</a>.</p></li> 
              <li><p>2010-08-17: <a href="http://meta.stackexchange.com/questions/11602/what-formula-should-be-used-to-determine-hot-questions" title="documented here">hot questions formula</a> has been <a href="http://meta.stackexchange.com/a/61343/165773" title="as described here">tweaked</a>: Succeeding questions from the same site are penalized by increasing amounts. Community wiki questions are penalized. The benefit of many answers is capped at 10, and we only look at the score of the top 3 answers. We only degrade based on question age, and not the last update date on a question.</p></li> 
              <li><p>2010-08-13: An <a href="http://meta.stackexchange.com/questions/4376/please-allow-image-uploads-directly-onto-the-site/60782#60782">image-upload capability</a> has been added, accessible through the image option in the toolbar.</p></li> 
              <li><p>2010-08-10: Questions from IP addresses or accounts with a history of <em>extremely</em> poor questions are <a href="http://meta.stackexchange.com/questions/56817/can-we-prevent-some-of-the-low-quality-questions-from-entering-our-system/60294#60294">no longer being accepted</a>. Votes on questions are taken into account, and this yields <a href="http://meta.stackexchange.com/questions/72180/clarify-no-longer-accepting-questions-from-this-account-error/76167#comment-174366">a permanent IP ban</a>, showing "Sorry, we are no longer accepting questions from this account".</p></li> 
              <li><p>2010-08-06: <a href="http://meta.stackexchange.com/questions/tagged/subjective" class="post-tag" title="show questions tagged 'subjective'" rel="tag">subjective</a> (and other meta-tags) <a href="http://meta.stackexchange.com/questions/51627/should-we-permanently-remove-the-subjective-tag">are now banned.</a></p></li> 
              <li><p>2010-08-03: Searching within favorites <a href="http://meta.stackexchange.com/questions/3683/search-in-favorited-question">is now included</a> in <a href="http://meta.stackoverflow.com/search">advanced search</a>.</p></li> 
             </ul> 
             <h3>Jul 2010</h3> 
             <ul> 
              <li><p>2010-07-22: "Info" tab of tags, including a <a href="http://meta.stackexchange.com/questions/1195/create-a-tag-homepage-faq/49723#49723">tag wiki</a> and a <a href="http://meta.stackexchange.com/questions/57479">tag synonym proposal system</a> implemented.</p></li> 
              <li><p>2010-07-17: <a href="http://meta.stackexchange.com/questions/51947/allow-users-to-opt-out-of-all-favorite-notifications/57550#57550">New user preference</a>: <em>I don't want notifications of activity on questions I've favorited</em> (to disable the notification that was introduced 2010-05-31).</p></li> 
              <li><p>2010-07-09: New badges for <a href="http://meta.stackexchange.com/questions/56067/bounty-badges-need-pretty-names">bounties</a> (<a href="http://meta.stackoverflow.com/badges/62/promoter">Promoter</a>, <a href="http://meta.stackoverflow.com/badges/68/benefactor">Benefactor</a>, <a href="http://meta.stackoverflow.com/badges/61/investor">Investor</a>, <a href="http://meta.stackoverflow.com/badges/64/altruist">Altruist</a>) and a <a href="http://meta.stackexchange.com/questions/1524/a-higher-level-strunk-white-badge-1000-edits-or-such">gold-level editor badge</a> (<a href="http://meta.stackoverflow.com/badges/65/copy-editor">Copy Editor</a>) introduced. Additionally, the <a href="http://meta.stackexchange.com/questions/88/find-out-which-question-answer-awarded-you-the-badge">origins of badges are now recorded</a>, and visible through clicking on a badge on the Badges page or a user's profile.</p></li> 
              <li><p>2010-07-09: Bounty award tooltips <a href="http://meta.stackexchange.com/questions/56111/mention-who-awarded-the-bounty">now display the number of bounties awarded and who awarded them</a>.</p></li> 
             </ul> 
             <h3>Jun 2010</h3> 
             <ul> 
              <li><p>2010-06-23: Using the browser's Back button towards the user page, now <a href="http://meta.stackexchange.com/questions/28584/back-arrow-to-recent-activity/55311#55311">scrolls that page back</a> to where it was.</p></li> 
              <li><p>2010-06-22: Your profile now <a href="http://meta.stackexchange.com/questions/865/consecutive-day-count-in-profile">tracks how many days you have visited the site, and how many consecutive days</a>. This is visible only to yourself and to diamond mods. No more worrying about whether you've missed a day!</p></li> 
              <li><p>2010-06-21: <a href="http://meta.stackexchange.com/questions/54366/feedback-request-newer-timeline-question-view">Timeline view for revision history introduced.</a> It has since been rolled out system-wide. <em>(date is approximate)</em></p></li> 
              <li><p>2010-06-19: <a href="http://blog.stackoverflow.com/2010/06/improvements-to-bounty-system/">Complete overhaul</a> of the bounty system. Any user with sufficient reputation can start a bounty on any question. A question may have multiple bounties, though only one active bounty is allowed at any given time. Bounty awards are no longer tied to accepted answer in any way. But the system no longer tosses in +50 bonus reputation to a bounty. (<a href="http://blog.stackoverflow.com/2010/06/improvements-to-bounty-system/">More rules</a>.)</p></li> 
              <li><p>2010-06-13: SEDE, <a href="http://blog.stackoverflow.com/2010/06/introducing-stack-exchange-data-explorer/">Stack Exchange Data Explorer</a>, which for SO also <em>"can be used as a permanently linkable tool for teaching general SQL and relational database concepts — we can be our own Northwind or Pubs database, when answering questions tagged [sql]"</em>. (Since December 2010, its URL is <a href="http://data.stackexchange.com" rel="nofollow">data.stackexchange.com</a>.)</p></li> 
              <li><p>2010-06-11: User profile <a href="http://meta.stackexchange.com/questions/48224/markdown-use-on-the-profile-page/53286#53286">is Markdown enabled</a>.</p></li> 
              <li><p>2010-06-09: <a href="http://blog.stackoverflow.com/2010/06/improved-question-merging/">Merged questions are no longer utterly annihilated</a>. A stub is left, just like with closing, and the revision list for the target question keeps better details on merges.</p></li> 
              <li><p>2010-06-07: <a href="http://meta.stackexchange.com/questions/52764/what-is-a-protected-question-how-do-they-work">Protected question</a> feature has been added so that 1 rep users can't post/spam on questions with a high search engine ranking.</p></li> 
              <li><p>2010-06-05: <a href="http://meta.stackexchange.com/questions/31088/reset-post-score-to-0-on-migration/52586#52586">Reset post score to 0 on migration</a>, if the score is negative. Automatic <em>301 Moved Permanently</em> <a href="http://meta.stackexchange.com/questions/17552/why-do-migrated-questions-lack-redirection-via-http-response-status-code-301-mov/52585#52585">redirects for migrated, deleted questions</a>, for users with &lt; 10k reputation. Add <code>noredirect=1</code> to avoid this.</p></li> 
              <li><p>2010-06-04: Signatures and taglines are <a href="http://meta.stackexchange.com/questions/5029/are-taglines-signatures-disallowed/5038#5038">officially discouraged</a> in the <code>/faq</code>s.</p></li> 
              <li><p>2010-06-03: <a href="http://meta.stackexchange.com/questions/50276/popup-alert-overlapping-bug/52244#52244">Shadows in popups</a>. New mechanism for <a href="http://meta.stackexchange.com/questions/50067/improve-close-as-exact-duplicate/52255#52255">displaying and selecting questions when voting to close as a duplicate</a>. Keyboard <a href="http://meta.stackexchange.com/questions/39490/could-we-close-the-why-should-this-question-be-closed-box-with-the-escape-key">Esc</a> can be used to close the windows.</p></li> 
              <li><p>2010-06-02: The Super User FAQ now explicitly prohibits <a href="http://meta.stackexchange.com/questions/51813/should-we-add-a-no-shopping-recommendation-clause-to-the-super-user-faq/52091#52091">shopping or buying recommendations</a>.</p></li>
