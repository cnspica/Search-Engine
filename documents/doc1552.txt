      <h1 itemprop="name"><a href="/questions/278889/custom-tag-styling-color-for-a-classification-of-tag" class="question-hyperlink">Custom tag styling (color) for a classification of tag?</a></h1>

            <div class="post-text" itemprop="text"> 
             <p>Tags come in (essentially) three colors... and these three colors are limited to meta sites only.</p> 
             <ul> 
              <li>Mod tags - red/maroon.</li> 
              <li>Required tags - grey.</li> 
              <li>General tags - white or nearly white depending on the site.</li> 
             </ul> 
             <p>As far as I know, all tags on a main site are the same color with the exception of the custom sponsored tags that some sites have, which have fun icons of the brand on them, like the Android tag on SO.</p> 
             <p>Some sites (not many, but some) have very specific types of tags. For example, on Movies &amp; TV, we have tags for film titles, Sci-Fi and Fantasy has tags for titles, authors, etc… Anime &amp; Manga is similar. </p> 
             <p>Occasionally, on M&amp;TV, we get users (usually 1 rep) who use generic-looking tags like <a href="/questions/tagged/aliens" class="post-tag" title="show questions tagged 'aliens'" rel="tag">aliens</a> to ask questions about… well… aliens in films… but the Aliens tag, as the tag wiki notes, is for questions about the film <em>Aliens</em>. Sometimes, if these tags are for less-well-known films or were recently added, they don't have excerpts at all… this, obviously, is a separate issue.</p> 
             <p>To add to the title specific uses above, this request was recently made here on Meta.SE:</p> 
             <p><a href="http://meta.stackexchange.com/questions/279254/should-we-make-country-tags-visually-distinct">Should we make country tags visually distinct?</a></p> 
             <blockquote> 
              <p>Some SE sites have questions that apply only to certain countries, or where answers should at least take laws or customs in the country in question into account. <strong>Example SE sites would be Workplace, Academia, Expats, Travel, Money or Law.</strong> These sites use country tags for this purpose.</p> 
             </blockquote> 
             <p>Changing the color of the group of tags would be a less intrusive solution to the request to add tiny flags to identify country tags (and it would be less work). The tags already have the country name on them, so all that is needed to set them apart visibly is a different color of tag (one color for all country tags, not one color per country). This would make all of the country tags on a site more visible while not interfering or watering down Stack Exchange's ability to sell the icons to interested parties.</p> 
             <hr> 
             <p>What I would like to propose is a specialized type of tag denoted with a different color that allows sites to do a couple of things:</p> 
             <ul> 
              <li>Makes it obvious that the tag is not for general use, particularly when the tag looks like it's generic (see the aliens example above).</li> 
              <li>Makes the tag stand out on a question as a form of classification.</li> 
              <li>As on Meta, these tags would have precedence over other tags and be listed first to make them more obvious.</li> 
              <li>As with the tags on meta, they would not be exempt from the five tag limit.</li> 
              <li>Potentially, this <strong>could</strong> allow a site to have more than one tag with the same name but that's not completely necessary. This could have its own set of problems but it might be something that could be worked around and help out a lot when you have issues where a book or film title or a brand is the same as a common subject matter. <em>This is something I think can be discussed separately rather than being a part of the initial implementation.</em></li> 
             </ul> 
             <p>There are certainly other benefits that I'm missing above.</p> 
             <p>To help implement these tags correctly, I recommend that the creation/reclassification of new tags should be initially limited to either mods or sufficiently high-rep users (10K+). For established sites, this will be quite an endeavor at the outset as there are likely hundreds of tags that will need to be classified.</p> 
             <p>As one would expect, this is something that should be implemented on a per-site basis and be something that the site has to show a particular need for and community support of before it is allowed.</p> 
             <p>As a note, on M&amp;TV we have recommended a plan to set film and TV tags apart by including the year of first release in the tag but this plan is on hold due to the issue with film/show title length often conflicting with the <a href="//meta.stackexchange.com/questions/209248/increase-limit-on-tag-length" rel="nofollow">25-character limit for tags</a>.</p> 
             <p>Note, I <strong>am not</strong> asking to <a href="http://meta.stackexchange.com/questions/78849/allow-custom-colorization-of-tags">Allow custom colorization of tags</a>, which relates to a user-setting to mark favorite tags.</p>

           <td class="answercell"> 
            <div class="post-text" itemprop="text"> 
             <p>When the subject came up on Workplace one proposal was to use the styling that, on meta, is used for mandatory tags (choose at least one of these four). I objected to <em>that</em> proposal because we don't want the UI to suggest to users that they <em>must</em> use one of these tags; some questions don't require it.</p> 
             <p>However, the red moderator tags that you brought up are a good example. There is no requirement that a (meta) post have one from that set, and users are used to seeing different tag colors.</p> 
             <p>So I think having a single styling that is <em>neither</em> the color of meta mandatory tags <em>nor</em> the color of moderator-only tags would be valuable. This styling might be color (I don't know how hard that is to implement) or perhaps a bold outline or even just bold text. SE would need to enable the functionality; it would then be up to individual sites to populate the set of tags to get this treatment. </p> 
             <p>I think we probably want moderators to designate which tags are in this set, because tag categorization affects the whole site. Moderators manage custom close reasons even though the community crafts them; I think this is similar. (I don't mean that there should be any restrictions on <em>creating</em> or <em>using</em> the tags, only on deciding which ones get the special styling.)</p> 
             <p>The special tags should be managed as a set, not by setting a bit on individual tags, because it's important that the <em>set</em> be cohesive and consistent.</p>

           <td class="answercell"> 
            <div class="post-text" itemprop="text"> 
             <p>This was actually suggested on <a href="http://meta.stackoverflow.com/a/321217">MSO</a>. Like you, we have a generic sounding tag (Processing) that actually refers to a specific thing. The proposal was that the color symbolize the difference between generic and specific tags.</p> 
             <p>Like I said then, it's a good idea, but it will be a huge amount of work, especially if this is something you think should happen elsewhere on the SE network (this isn't M&amp;TV Meta, after all).</p> 
             <p>Not only are there an enormous amount of tags, there are an enormous number of tags that are very poorly defined. There is already plenty of arguments when retags are proposed, but that's usually only a handful of tags.</p> 
             <p>What you suggest will likely involve dozens, if not hundreds of changes to tag wikis on each site. Not to mention that there are plenty of umbrella tags that are used in both contexts. Many tags have no wiki whatsoever.</p> 
             <p>There's also a lot of work that this would mean for the developers. I expect that we all realized that someone will need to design and program this feature.</p> 
             <p>The design is an important thing to think about. Not only is color highly cultural, <a href="https://en.wikipedia.org/wiki/Colorless_green_ideas_sleep_furiously" rel="nofollow">there is no real color tied to ideas</a>. Many sites cater to a diverse audience, so it's an important thing to consider. Also, how will you pick a color without upsetting all the people who paid for tag icons? Changing the color of the non-generic tags runs the risk of clashing with icons like that.</p> 
             <hr> 
             <p>Currently, the Processing tag isn't abused at all on SO. A single user (the OP of that question I linked) <em>single handedly</em> keeps the tag clean. And SO has the most users of any site here, with plenty of users that just throw tags around.</p> 
             <p>The easiest solution is to just retag the questions yourself, and maintain it. Do the experienced members at least realize how to use the tag?</p> 
             <p>I would not get your hopes up and expect the new users to realize what tags mean, regardless of what color they may be.</p> 
             <p>I realize that this probably isn't what you want to hear, but I'm just trying to describe the reality of the situation, as I see it.</p>
