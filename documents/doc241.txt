             <p>They're called <a href="http://en.wikipedia.org/wiki/Identicon" rel="nofollow">Identicons</a>.</p> 
             <p>If you do not <a href="http://blog.stackoverflow.com/2013/01/announcing-a-new-way-to-change-your-profile-picture/">upload your own image</a>, then Stack Exchange uses Gravatar and specifies Identicons as the default image:</p> 
             <ul> 
              <li><p><a href="http://en.gravatar.com/site/implement/" rel="nofollow">Gravatar implementor's guide</a></p></li> 
              <li><p><a href="http://en.gravatar.com/site/implement/hash/" rel="nofollow">How the URL is constructed</a>, with:</p> 
               <ul> 
                <li><p>When you're not registered (when Stack Exchange does not know your email address), this is based on your last known IP address so <a href="http://meta.stackexchange.com/questions/37762/how-does-an-identicon-identify-a-user/37763#37763">might change when your IP address changes</a></p></li> 
                <li><p>For new users and those who click the "Change image" in their profile, <a href="http://meta.stackexchange.com/questions/204523/unexpectedly-changing-identicon/204540#204540">Stack Exchange adds a salt if the user does not have a Gravatar account</a>, to make it a bit more secure</p></li> 
               </ul></li> 
              <li><p><a href="https://web.archive.org/web/20080703155519/www.docuverse.com/blog/donpark/2007/01/18/visual-security-9-block-ip-identification" rel="nofollow">Original blog post by Don Park, the Identicons inventor</a>; their <a href="https://github.com/donpark/identicon" rel="nofollow">implementation in Java</a> is available still.</p></li> 
              <li><p><a href="http://scott.sherrillmix.com/blog/blogger/wp_identicon/" rel="nofollow">Blog post by the creators of a PHP implementation of Identicons</a> as used by Gravatar.</p></li> 
             </ul>

             <p>Checkout <a href="http://blog.codinghorror.com/identicons-for-net/" rel="nofollow">Coding Horror</a>, it has an algorithm based on IP Address similar to SO.</p> 
             <p>From <a href="http://en.wikipedia.org/wiki/Identicon" rel="nofollow">Wikipedia:</a></p> 
             <blockquote> 
              <p>An Identicon is a visual representation of a hash value, usually of the IP address, serving to identify a user of a computer system. The original Identicon is a 9-block graphic, which has been extended to other graphic forms by third parties some of whom have used MD5 instead of the IP address as the identifier. In summary, an Identicon is a privacy protecting derivative of each user's IP address built into a 9-block image and displayed next the user's name. A visual representation is thought to be easier to compare than one which uses only numbers and more importantly, it maintains the person's privacy. The Identicon graphic is unique since it's based on the users IP, but it is not possible to recover the IP by looking at the Identicon.</p> 
             </blockquote>

            <div class="post-text" itemprop="text"> 
             <p><em>As regards showing them automatically with Gravatar, here is the answer I posted <a href="http://stackoverflow.com/questions/405717/how-to-auto-random-generate-the-gravatars-like-on-stackoverflow#405723">here</a>:</em></p> 
             <p>The 'random' colorful gravatars are displayed because this query string parameter is being added to every gravatar source url: <code>d=identicon</code></p> 
             <p>This is done so that if the user doesn't have a gravatar image associated with his email, this 'random' image is displayed, instead of the default blue gravatar image.</p> 
             <p>The following displays the 'default' blue image because the parameter is not included: <img src="http://www.gravatar.com/avatar/94d093eda664addd6e450d7e9881bcad?s=32&amp;r=PG" alt="alt text"></p> 
             <p>Yet, the same url with the <code>d=identicon</code> parameter included, shows this: <img src="http://www.gravatar.com/avatar/94d093eda664addd6e450d7e9881bcad?s=32&amp;d=identicon&amp;r=PG" alt="alt text"></p> 
             <p><em>PS: This is the url used for the example: <a href="http://www.gravatar.com/avatar/94d093eda664addd6e450d7e9881bcad?s=32&amp;d=identicon&amp;r=PG" rel="nofollow">http://www.gravatar.com/avatar/94d093eda664addd6e450d7e9881bcad?s=32&amp;d=identicon&amp;r=PG</a></em></p> 
             <p>Btw, these images (called <a href="http://en.wikipedia.org/wiki/Identicon" rel="nofollow">Identicons</a>) are not really random, but are generated based on a Hash of your IP address.</p>
