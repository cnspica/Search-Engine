             <p>If you play the file by dragging and dropping, pause or stop it, then:</p> 
             <ul> 
              <li>go to play-list,</li> 
              <li>right-click on the problem file, and</li> 
              <li>click <code>Information...</code></li> 
             </ul> 
             <p>The <code>Codec</code> tab will tell you what kind of file it is, and you can now exit and rename the file to add an appropriate extension.</p> 
             <p><strong>Don't convert the file!</strong> This will take time and degrade the quality. If you don't know what the appropriate extension should be, rename it to <code>FileName.vlc</code> and associate this extension with VLC (any unused extension could be used, but this one is unlikely to be used by anything else).</p>

           <td class="answercell"> 
            <div class="post-text" itemprop="text"> 
             <p>If you absolutely <em>must</em> convert the file to mp3, I would recommend <a href="http://ffmpeg.org/" rel="nofollow">ffmpeg</a>.</p> 
             <p>You can convert your file on the command line:</p> 
             <p><code>ffmpeg -i random-audio-file -c:a libmp3lame -q:a 4 foobar.mp3</code></p> 
             <p>But as stated by others, if VLC can already play it, there's no need to convert it (and thus lose quality).</p> 
             <p><strong>It's better to identify the filetype or codec, and rename the file to the appropriate extension which you can then bind in Windows to open with VLC by default.</strong></p> 
             <p>To identify the filetype or audio format, you can use <code>ffprobe</code> (which is also part of ffmpeg). Or use various other tools such as <a href="https://mediaarea.net/en/MediaInfo" rel="nofollow">MediaInfo</a> which is perhaps more user friendly, especially for Windows users.</p>

            <div class="post-text" itemprop="text"> 
             <h2>Blind, quick and dirty.</h2> 
             <p>You stated that yours is an mp3 file without the extension (vlc played it).<br> <strong>You have not to convert it, you have simply to rename it adding the extension <code>.mp3</code>.</strong><br> If you convert it you will loose something in term of quality. </p> 
             <p>Note that even if it was not an <em>mp3 file</em> probably each version of VLC will play it anyway even if you rename it with a wrong extension. The problem, if present, will remain but you will not be aware... </p> 
             <hr> 
             <p><sup>If you feel unconfortable reading the above answer maybe you want to read some lines more... </sup><br> ... A <em>wiser</em> way to proceed is to <strong>recognise</strong> which kind of file it is and which tools can help you...<br> but <strong>this is another <em>quest</em>(ion)</strong>, many time asked, answered and closed before <sup>[<a href="http://superuser.com/questions/274734/is-there-a-tool-that-can-determine-the-file-type-from-containing-data">q1</a>],[<a href="http://superuser.com/questions/435224/how-do-i-find-out-the-file-type-without-an-extension">q2</a>]</sup> <code>:-)</code>. </p> 
             <p>It is often possible because many standards require identification codes in specific parts of the file, usually in the beginning. That is the reason because vlc can play even a part of a file (not yet fully downloaded).</p> 
             <p>More you know about the file more ideas car arise after. So what are your options? </p> 
             <ul> 
              <li><strong>Vlc</strong> can do it, as suggested in another good answer <sup>[<a href="http://superuser.com/a/1075394/257269">1</a>]</sup>. </li> 
              <li>If you have many files, it can be time consuming to repeat with a GUI that operation each time.<br> <sup> Many tools, as for example <strong>TrID</strong> <sup>[<a href="http://mark0.net/soft-trid-e.html" rel="nofollow">2</a>]</sup>, can identify them for you and eventually automatically rename all the files.</sup></li> 
              <li><p>Unfortunately, windows 7 by default do not include the gnu <strong><code>file</code></strong> command <sup>[<a href="http://gnuwin32.sourceforge.net/packages/file.htm" rel="nofollow">3</a>]</sup>. But fortunately you can download and install it. Then from a shell (execute the program <code>cmd.exe</code>), you can arrive to your file (<code>cd yourpath</code>) then write the command </p> <pre><code>file MyUnknowFile
</code></pre> <p>and it will answer with something like</p> 
               <blockquote> 
                <p>Audio file with ID3 version 2.3.0, contains: <strong>MPEG</strong> ADTS, <strong>layer III</strong>, v1, 192 kbps, 44.1 kHz, Stereo</p> 
               </blockquote> <p>I put in bold <strong>MPEG</strong> and <strong>layer III</strong>... so this is an <strong>mp3</strong> file and this is the extension that your file has somehow lost. You have only to add it to your file. Do you remember <em>"more you know ... more ideas"</em>? I can read <code>with ID3</code> and that gives me another idea (see below).</p></li> 
              <li>You can use an <strong>ID3 tag <em>renamer</em></strong> <sup>[<a href="http://superuser.com/search?q=rename+id3">q3</a>]</sup>, yes you can change not only the extension but even the <em>filename</em> compliant with the information included in the ID3 tag !</li> 
              <li>You can use an <a href="http://www.1mtb.com/top-5-best-web-based-online-music-song-identification-services/" rel="nofollow">online service</a> to identify the song. Some of them are able to overcome the wrong extension too...</li> 
             </ul> 
             <p>But the path of an heroic champion pass through the hard way. The way of the paladin is to see the binary (maybe the hexadecimal) code of your file in search of the <em>magic numbers</em> <sup>[<a href="https://en.wikipedia.org/wiki/Magic_number_%28programming%29" rel="nofollow">w1</a>]</sup> looking in a <em>reference table</em> <sup>[<a href="https://en.wikipedia.org/wiki/List_of_file_signatures" rel="nofollow">w2</a>]</sup> to understand which file is... and eventually re-invent the wheel. <code>:-)</code>. </p>
