        <p>Any value written within a pair of single quote or double quotes in R is treated as a string. Internally R stores every string within double quotes, even when you create them with single quote.</p> 
        <h2>Rules Applied in String Construction</h2> 
        <ul class="list"> 
         <li><p>The quotes at the beginning and end of a string should be both double quotes or both single quote. They can not be mixed.</p></li> 
         <li><p>Double quotes can be inserted into a string starting and ending with single quote.</p></li> 
         <li><p>Single quote can be inserted into a string starting and ending with double quotes.</p></li> 
         <li><p>Double quotes can not be inserted into a string starting and ending with double quotes.</p></li> 
         <li><p>Single quote can not be inserted into a string starting and ending with single quote.</p></li> 
        </ul> 
        <h3>Examples of Valid Strings</h3> 
        <p>Following examples clarify the rules about creating a string in R.</p> 
        <pre class="prettyprint notranslate tryit">
a &lt;- 'Start and end with single quote'
print(a)

b &lt;- "Start and end with double quotes"
print(b)

c &lt;- "single quote ' in between double quotes"
print(c)

d &lt;- 'Double quotes " in between single quote'
print(d)
</pre> 
        <p>When the above code is run we get the following output −</p> 
        <pre class="result notranslate">
[1] "Start and end with single quote"
[1] "Start and end with double quotes"
[1] "single quote ' in between double quote"
[1] "Double quote \" in between single quote"
</pre> 
        <h3>Examples of Invalid Strings</h3> 
        <pre class="prettyprint notranslate tryit">
e &lt;- 'Mixed quotes" 
print(e)

f &lt;- 'Single quote ' inside single quote'
print(f)

g &lt;- "Double quotes " inside double quotes"
print(g)
</pre> 
        <p>When we run the script it fails giving below results.</p> 
        <pre class="result notranslate">
...: unexpected INCOMPLETE_STRING

.... unexpected symbol 
1: f &lt;- 'Single quote ' inside

unexpected symbol
1: g &lt;- "Double quotes " inside
</pre> 
        <h2>String Manipulation</h2> 
        <h3>Concatenating Strings - paste() function</h3> 
        <p>Many strings in R are combined using the <b>paste()</b> function. It can take any number of arguments to be combined together.</p> 
        <h3>Syntax</h3> 
        <p>The basic syntax for paste function is −</p> 
        <pre class="result notranslate">
paste(..., sep = " ", collapse = NULL)
</pre> 
        <p>Following is the description of the parameters used −</p> 
        <ul class="list"> 
         <li><p><b>...</b> represents any number of arguments to be combined.</p></li> 
         <li><p><b>sep</b> represents any separator between the arguments. It is optional.</p></li> 
         <li><p><b>collapse</b> is used to eliminate the space in between two strings. But not the space within two words of one string.</p></li> 
        </ul> 
        <h3>Example</h3> 
        <pre class="prettyprint notranslate tryit">
a &lt;- "Hello"
b &lt;- 'How'
c &lt;- "are you? "

print(paste(a,b,c))

print(paste(a,b,c, sep = "-"))

print(paste(a,b,c, sep = "", collapse = ""))
</pre> 
        <p>When we execute the above code, it produces the following result −</p> 
        <pre class="result notranslate">
[1] "Hello How are you? "
[1] "Hello-How-are you? "
[1] "HelloHoware you? "
</pre> 
        <h3>Formatting numbers &amp; strings - format() function</h3> 
        <p>Numbers and strings can be formatted to a specific style using <b>format()</b> function.</p> 
        <h3>Syntax</h3> 
        <p>The basic syntax for format function is −</p> 
        <pre class="result notranslate">
format(x, digits, nsmall, scientific, width, justify = c("left", "right", "centre", "none")) 
</pre> 
        <p>Following is the description of the parameters used −</p> 
        <ul class="list"> 
         <li><p><b>x</b> is the vector input.</p></li> 
         <li><p><b>digits</b> is the total number of digits displayed.</p></li> 
         <li><p><b>nsmall</b> is the minimum number of digits to the right of the decimal point.</p></li> 
         <li><p><b>scientific</b> is set to TRUE to display scientific notation.</p></li> 
         <li><p><b>width</b> indicates the minimum width to be displayed by padding blanks in the beginning.</p></li> 
         <li><p><b>justify</b> is the display of the string to left, right or center.</p></li> 
        </ul> 
        <h3>Example</h3> 
        <pre class="prettyprint notranslate tryit">
# Total number of digits displayed. Last digit rounded off.
result &lt;- format(23.123456789, digits = 9)
print(result)

# Display numbers in scientific notation.
result &lt;- format(c(6, 13.14521), scientific = TRUE)
print(result)

# The minimum number of digits to the right of the decimal point.
result &lt;- format(23.47, nsmall = 5)
print(result)

# Format treats everything as a string.
result &lt;- format(6)
print(result)

# Numbers are padded with blank in the beginning for width.
result &lt;- format(13.7, width = 6)
print(result)

# Left justify strings.
result &lt;- format("Hello", width = 8, justify = "l")
print(result)

# Justfy string with center.
result &lt;- format("Hello", width = 8, justify = "c")
print(result)
</pre> 
        <p>When we execute the above code, it produces the following result −</p> 
        <pre class="result notranslate">
[1] "23.1234568"
[1] "6.000000e+00" "1.314521e+01"
[1] "23.47000"
[1] "6"
[1] "  13.7"
[1] "Hello   "
[1] " Hello  "
</pre> 
        <h3>Counting number of characters in a string - nchar() function</h3> 
        <p>This function counts the number of characters including spaces in a string.</p> 
        <h3>Syntax</h3> 
        <p>The basic syntax for nchar() function is −</p> 
        <pre class="result notranslate">
nchar(x)
</pre> 
        <p>Following is the description of the parameters used −</p> 
        <ul class="list"> 
         <li><p><b>x</b> is the vector input.</p></li> 
        </ul> 
        <h3>Example</h3> 
        <pre class="prettyprint notranslate tryit">
result &lt;- nchar("Count the number of characters")
print(result)
</pre> 
        <p>When we execute the above code, it produces the following result −</p> 
        <pre class="result notranslate">
[1] 30
</pre> 
        <h3>Changing the case - toupper() &amp; tolower() functions</h3> 
        <p>These functions change the case of characters of a string.</p> 
        <h3>Syntax</h3> 
        <p>The basic syntax for toupper() &amp; tolower() function is −</p> 
        <pre class="result notranslate">
toupper(x)
tolower(x)
</pre> 
        <p>Following is the description of the parameters used −</p> 
        <ul class="list"> 
         <li><p><b>x</b> is the vector input.</p></li> 
        </ul> 
        <h3>Example</h3> 
        <pre class="prettyprint notranslate tryit">
# Changing to Upper case.
result &lt;- toupper("Changing To Upper")
print(result)

# Changing to lower case.
result &lt;- tolower("Changing To Lower")
print(result)
</pre> 
        <p>When we execute the above code, it produces the following result −</p> 
        <pre class="result notranslate">
[1] "CHANGING TO UPPER"
[1] "changing to lower"
</pre> 
        <h3>Extracting parts of a string - substring() function</h3> 
        <p>This function extracts parts of a String.</p> 
        <h3>Syntax</h3> 
        <p>The basic syntax for substring() function is −</p> 
        <pre class="result notranslate">
substring(x,first,last)
</pre> 
        <p>Following is the description of the parameters used −</p> 
        <ul class="list"> 
         <li><p><b>x</b> is the character vector input.</p></li> 
         <li><p><b>first</b> is the position of the first character to be extracted.</p></li> 
         <li><p><b>last</b> is the position of the last character to be extracted.</p></li> 
        </ul> 
        <h3>Example</h3> 
        <pre class="prettyprint notranslate tryit">
# Extract characters from 5th to 7th position.
result &lt;- substring("Extract", 5, 7)
print(result)
</pre> 
        <p>When we execute the above code, it produces the following result −</p> 
        <pre class="result notranslate">
[1] "act"
</pre>
