            <div class="post-text" itemprop="text"> 
             <p>The following is my first successful running C# program.</p> 
             <p>The lessons that I personally learnt through this include:</p> 
             <ul> 
              <li>The value of asking for help after having tried several times</li> 
              <li>The advantages of being adaptable about one's methods of reaching an end</li> 
              <li>The (a?) proper placement of functions in relation to a main program</li> 
              <li>How to pass information into and out of such functions</li> 
              <li>How to use a loop to leverage code (doing the same stuff over and over)</li> 
              <li>How to collect character input from the User &amp; I'm sure there's more...</li> 
             </ul> 
             <p>In any case this code appears to work. I know that it doesn't hold a candle to much of the stuff you guys put up but on the bright side its short and kind of readable (both aims of mine).</p> 
             <p>Could some much brighter spark than me kindly shine a light on the stuff that could have been done better (or simply provide some advice or suggested direction)? :c)</p> 
             <pre><code>using System;
namespace CapsChecker
{
    class Program
    {
        static void Main(string[] args)
        {
                //Set up an infinite loop for program to run within 
            while (true)
            {
                    //Collection of user input
                Console.Write("Press a key: ");
                char input = (Console.ReadKey().KeyChar);
                    //Check if Capslock is on when key entered. Combined with 
                    //the shift button this can result in CapsCheck returning
                    //true even if key entered is lower case.
                if (CapsCheck() == true)
                {
                    CapsNotify(true);   
                }
                else
                {
                    CapsNotify(false);
                }
                Console.WriteLine();

                switch (input) //Abandoned '.Key' enum
                {
                    case 'Q':
                        { break; } 
                    default:
                        { continue; }
                }
                break; //Afterthoght as it wasn't breaking on capital Q.
            }
            Console.WriteLine("Program ending. Press a key.");
            Console.ReadKey();
        }

        static bool CapsCheck()
        {
                // Check if Capslock is Active
            if (Console.CapsLock == true)
            {
                return true;
            }
            else
            {
                return false;
            }
        }

        static void CapsNotify(bool caps)
        {
                // Notify whether Capslock is Active
            if (caps == true)
            {
                Console.WriteLine(" Capslock is Active.");
            }
            else
            {
                Console.WriteLine(" Capslock is Inactive.");
            }
        }
    }
}
</code></pre>

            <div class="post-text" itemprop="text"> 
             <h3><code>if true return true else return false</code> antipattern</h3> 
             <p>Unecessary <code>if</code> statements when dealing with booleans is a very common beginner pitfall.</p> 
             <pre><code>    static bool CapsCheck()
    {
            // Check if Capslock is Active
        if (Console.CapsLock == true)
        {
            return true;
        }
        else
        {
            return false;
        }
    }
</code></pre> 
             <p>Is equivalent to:</p> 
             <pre><code>   static bool CapsCheck()
   {
       return Console.CapsLock;
   }
</code></pre> 
             <p>As a further simplification, you may use <code>Console.CapsLock</code> directly and avoid this function.</p> 
             <p>Another boolean simplification is</p> 
             <pre><code>            if (CapsCheck() == true)
            {
                CapsNotify(true);   
            }
            else
            {
                CapsNotify(false);
            }
</code></pre> 
             <p>To:</p> 
             <pre><code> CapsNotify( CapsCheck() );
</code></pre> 
             <p>Or:</p> 
             <pre><code>CapsNotify( Console.CapsLock );
</code></pre> 
             <h3>Ternary simplification</h3> 
             <p>In <code>CapsNotify</code> it is unecessary to use a full <code>if</code> <code>else</code>, a ternary is enough and simpler:</p> 
             <pre><code> Console.Writeline("CapsLock is" + (caps ? "Active" : "Inactive"));
</code></pre>

            <div class="post-text" itemprop="text"> 
             <p>Instead of using a forever loop with a <code>break</code>, which as you found out, <a href="http://stackoverflow.com/questions/4044752/breaking-out-of-a-foreach-loop-from-within-a-switch-block">gets complicated when there's a switch involved</a>, evaluate your end condition inside of your while. </p> 
             <p>So instead of</p> 
             <pre><code>while(true)
{
    //...

     switch (input) //Abandoned '.Key' enum
     {
        case 'Q':
           { break; } 
        default:
            { continue; }
      }
      break; //Afterthoght as it wasn't breaking on capital Q.
        }
</code></pre> 
             <p>You can just do this</p> 
             <pre><code>char input = default(char);

do
{

    //...

} while (input != 'Q');
</code></pre> 
             <p>Because you always want your main loop to execute at least once. </p> 
             <p>See the <a href="https://msdn.microsoft.com/en-us/library/370s1zax.aspx" rel="nofollow">msdn doc for <code>do...while</code> loops</a> for more information. </p>
