  <summary type="html"> &lt;p&gt;I think it is very frustrating and hurts the sites "usefulness" that people are so promt at closing topics they don't really understand.&lt;/p&gt; &lt;p&gt;It would be nice solutions were found avoid theses cases:&lt;/p&gt; &lt;ul&gt; &lt;li&gt;Legitimate cross-domain questions The question can be related to domain A and B. On A.stackexchange.com, people say it's off-topic and you should ask B.stackexchange.com. On B.stackexchange.com, people say it's off-topic and you should ask A.stackexchange.com.&lt;/li&gt; &lt;/ul&gt; &lt;p&gt;Real example: Ask a question about photography software. "Software" will say that only photographers use that specific software so you should ask "Photography". "Photography" will say that only software geeks know how to use software so you should ask "Software".&lt;/p&gt; &lt;p&gt;I think if it is a legitimate cross-domain question, the same question should be able to appear in both sites without cross-posting. By the way this should required some validation. I know, that's not a small feature change.&lt;/p&gt; &lt;ul&gt; &lt;li&gt;Sub-domain topics The question is about topic X in domain A. People who know a lot about domain A but don't know topic X will downvote the question or tag it as off-topic.&lt;/li&gt; &lt;/ul&gt; &lt;p&gt;Those happen even if the [domain].stackexchange.com site has a dedicated tag for topic X.&lt;/p&gt; &lt;p&gt;As a result, specific questions tend to get closed by users who don't know the topic and therefore are not given a chance a to reach the small fraction of user who could answer them.&lt;/p&gt; &lt;p&gt;Or maybe I missed something?&lt;/p&gt; </summary>

For example, on Movies &amp;amp; TV, we have tags for film titles, Sci-Fi and Fantasy has tags for titles, authors, etc… Anime &amp;amp; Manga is similar. &lt;/p&gt; &lt;p&gt;Occasionally, on M&amp;amp;TV, we get users (usually 1 rep) who use generic-looking tags like &lt;a href="/questions/tagged/aliens" class="post-tag" title="show questions tagged &amp;#39;aliens&amp;#39;" rel="tag"&gt;aliens&lt;/a&gt; to ask questions about… well… aliens in films… but the Aliens tag, as the tag wiki notes, is for questions about the film &lt;em&gt;Aliens&lt;/em&gt;. Sometimes, if these tags are for less-well-known films or were recently added, they don't have excerpts at all… this, obviously, is a separate issue.&lt;/p&gt; &lt;p&gt;To add to the title specific uses above, this request was recently made here on Meta.SE:&lt;/p&gt; &lt;p&gt;&lt;a href="http://meta.stackexchange.com/questions/279254/should-we-make-country-tags-visually-distinct"&gt;Should we make country tags visually distinct?&lt;/a&gt;&lt;/p&gt; &lt;blockquote&gt; &lt;p&gt;Some SE sites have questions that apply only to certain countries, or where answers should at least take laws or customs in the country in question into account. &lt;strong&gt;Example SE sites would be Workplace, Academia, Expats, Travel, Money or Law.&lt;/strong&gt; These sites use country tags for this purpose.&lt;/p&gt; &lt;/blockquote&gt; &lt;p&gt;Changing the color of the group of tags would be a less intrusive solution to the request to add tiny flags to identify country tags (and it would be less work). The tags already have the country name on them, so all that is needed to set them apart visibly is a different color of tag (one color for all country tags, not one color per country). This would make all of the country tags on a site more visible while not interfering or watering down Stack Exchange's ability to sell the icons to interested parties.&lt;/p&gt; &lt;hr&gt; &lt;p&gt;What I would like to propose is a specialized type of tag denoted with a different color that allows sites to do a couple of things:&lt;/p&gt; &lt;ul&gt; &lt;li&gt;Makes it obvious that the tag is not for general use, particularly when the tag looks like it's generic (see the aliens example above).&lt;/li&gt; &lt;li&gt;Makes the tag stand out on a question as a form of classification.&lt;/li&gt; &lt;li&gt;As on Meta, these tags would have precedence over other tags and be listed first to make them more obvious.&lt;/li&gt; &lt;li&gt;As with the tags on meta, they would not be exempt from the five tag limit.&lt;/li&gt; &lt;li&gt;Potentially, this &lt;strong&gt;could&lt;/strong&gt; allow a site to have more than one tag with the same name but that's not completely necessary. This could have its own set of problems but it might be something that could be worked around and help out a lot when you have issues where a book or film title or a brand is the same as a common subject matter. &lt;em&gt;This is something I think can be discussed separately rather than being a part of the initial implementation.&lt;/em&gt;&lt;/li&gt; &lt;/ul&gt; &lt;p&gt;There are certainly other benefits that I'm missing above.&lt;/p&gt; &lt;p&gt;To help implement these tags correctly, I recommend that the creation/reclassification of new tags should be initially limited to either mods or sufficiently high-rep users (10K+). For established sites, this will be quite an endeavor at the outset as there are likely hundreds of tags that will need to be classified.&lt;/p&gt; &lt;p&gt;As one would expect, this is something that should be implemented on a per-site basis and be something that the site has to show a particular need for and community support of before it is allowed.&lt;/p&gt; &lt;p&gt;As a note, on M&amp;amp;TV we have recommended a plan to set film and TV tags apart by including the year of first release in the tag but this plan is on hold due to the issue with film/show title length often conflicting with the &lt;a href="//meta.stackexchange.com/questions/209248/increase-limit-on-tag-length" rel="nofollow"&gt;25-character limit for tags&lt;/a&gt;.&lt;/p&gt; &lt;p&gt;Note, I &lt;strong&gt;am not&lt;/strong&gt; asking to &lt;a href="http://meta.stackexchange.com/questions/78849/allow-custom-colorization-of-tags"&gt;Allow custom colorization of tags&lt;/a&gt;, which relates to a user-setting to mark favorite tags.&lt;/p&gt; </summary>
