            <div class="post-text" itemprop="text"> 
             <p>Currently, to see <code>console.log()</code> entries from StackSnippets, you need to open your browser console, and make sure it's recording from frames. Things could be a lot simpler if you would add a console to the snippets. Maybe something at the bottom of the snippet.</p> 
             <p>This could be done by overwriting the default <code>console.log</code> function with a custom one that both writes to the browser console (keep a reference to the original function), and also writes to the snippet's console.</p> 
             <p>EDIT:<br> And perhaps also show errors thrown in this console.</p>

           <td class="answercell"> 
            <div class="post-text" itemprop="text"> 
             <p>A duplicate of this question came up <a href="http://meta.stackoverflow.com/q/273049/621962">on meta.SO</a> where I suggested <a href="http://meta.stackoverflow.com/a/273249/621962">a solution</a>, copied here: </p>
             <hr> I whole-heartedly agree. If the point of these snippets is to enable us to demo functionality and display its output inline, the output pane needs to be adapted to (optionally) mirror logged messages. 
             <p></p> 
             <p>Why should we bother avoiding <code>document.write()</code> and <code>alert()</code>?</p> 
             <ul> 
              <li>Both assume a browser environment even if we're demonstrating a purely javascript concept. We certainly don't want either method in the middle of what should be a node.js function. </li> 
              <li>Neither are what we're using to debug (I hope)</li> 
              <li><code>alert()</code> is extremely intrusive.</li> 
              <li><code>document.write()</code> shouldn't even be suggested to the novice user without a littany of caveats</li> 
              <li>Neither method offers robust serialization... forget serializing objects.</li> 
              <li>Our debugging messages deserve more love than that from our demo tool. We can do much, much better.</li> 
             </ul> 
             <h1>My Temporary Solution</h1> 
             <p>I've created a small script to include via the html snippet<sup>1</sup> (please leave the comment intact):</p> 
             <pre class="lang-html prettyprint-override"><code>&lt;!-- console visualization; see http://meta.stackexchange.com/a/242491 --&gt;
&lt;script src="https://gh-canon.github.io/stack-snippet-console/console.min.js"&gt;&lt;/script&gt;
</code></pre> 
             <p><sub><sup>1</sup> If we ever get integrated console visualization, we can just remove this script wholesale and it should be very low-impact.</sub></p> 
             <p>Notes:</p> 
             <ul> 
              <li>All calls to <code>console.log()</code> (<code>info()</code>, <code>warn()</code>, <code>error()</code>, and <code>clear()</code>) are passed along to the actual <code>console</code></li> 
              <li>Loosely imitates Chrome's output (colors and styles excluded)</li> 
              <li>Objects are printed using a variation of <code>JSON.stringify()</code> (including functions, regex, undefined, etc) with custom handling for <code>HTMLElement</code></li> 
              <li>Works by injecting elements into the page... so, it obviously pollutes the DOM.</li> 
              <li>The number of entries is limited to 50. Older entries will be removed to make room for newer entries.</li> 
              <li>Logs unhandled errors.</li> 
              <li>Handles circular references. You could log <code>window</code>; though, I wouldn't recommend it.</li> 
             </ul> 
             <p>Here's a demo:</p> 
             <p></p>
             <div class="snippet" data-lang="js" data-hide="false"> 
              <div class="snippet-code"> 
               <pre class="snippet-code-js lang-js prettyprint-override"><code>// sample values
var values = [1e100, "a", true, new Date(), x =&gt; 1, /(?:)/ig, undefined, null];

// log whole array
console.log("whole array: %o", values);

// add a circular reference
values.push(values);

// log whole array 
console.log("array with circular reference: %o", values);

// remove circular reference
values.pop();

// log each value individually
values.forEach((v, i) =&gt; console.log("%i: %o", i, v));

// sample click handler
document.querySelector("button").addEventListener("click", function(e) {
  console.log("clicked: %o", this);
});

// throw a test error
throw new Error("test!");</code></pre> 
               <pre class="snippet-code-html lang-html prettyprint-override"><code>&lt;!-- console visualization; see http://meta.stackexchange.com/a/242491 --&gt;
&lt;script src="https://gh-canon.github.io/stack-snippet-console/console.min.js"&gt;&lt;/script&gt;
&lt;button&gt;click me&lt;/button&gt;</code></pre> 
              </div>

            <div class="post-text" itemprop="text"> 
             <p><strong>TL;DR</strong></p> 
             <ol> 
              <li><p>Best to leave the actual console alone, not least because interfering with it can prevent asking and answering questions related to the console (such as when and how data is expanded on V8's console, which is a non-trivial thing).</p></li> 
              <li><p>Snippets definitely need a built-in, simple means of generating output without promoting poor practices like <code>document.write</code>.</p></li> 
              <li><p>Specific suggestion for what to provide below, including <a href="http://tjcrowder.github.io/simple-snippets-console/snippet.js" rel="nofollow">a version hosted on GitHub Pages</a> for use until/unless Snippets have their own way.</p></li> 
             </ol> 
             <hr> 
             <p>I think the <em>console</em> should be left alone, not least because interfering with it can prevent asking and answering questions related to the console (such as when and how data is expanded on V8's console, which is a non-trivial thing).</p> 
             <p>But I'm <strong><em>very</em></strong> keen to see what I think is the main point of this request: A simple means of writing output to the output pane of the snippet.</p> 
             <p>Right now, people are doing four things:</p> 
             <ol> 
              <li><p>Using <code>document.write</code>. This is <strong>awful</strong>, it promotes 1990s code in 2014+.</p></li> 
              <li><p>Using some function for output, like my usual:</p> <pre class="lang-js prettyprint-override"><code>function display(msg) {
    document.body.insertAdjacentHTML(
        "beforeend",
        "&lt;p&gt;" + msg + "&lt;/p&gt;"
    );
}
</code></pre> <p>This is also <strong>not good</strong>, because it adds a lot of noise, esp. to short snippets.</p></li> 
              <li><p>Using <code>console.log</code>, which just makes the output pane of the snippet wasted space.</p></li> 
              <li><p>Writing scripts they include in snippets (like mine below), which means the snippet relies on an off-site resource.</p></li> 
             </ol> 
             <p>None of those is a good solution.</p> 
             <p>My suggestion is <strong>Keep It Simple</strong> and inject a <code>snippet</code> object into snippets (<em>without</em> showing it in the snippet) with a <code>log</code> (or whatever) method; perhaps both <code>log</code> and <code>logHTML</code>:</p> 
             <pre class="lang-js prettyprint-override"><code>// MIT license, see: https://github.com/tjcrowder/simple-snippets-console/blob/master/LICENSE
var snippet = {
    version: "1.3",

    // Writes out the given text in a monospaced paragraph tag, escaping
    // &amp; and &lt; so they aren't rendered as HTML.
    log: function(msg, tag) {
        var elm = document.createElement(tag || "p");
        elm.style.fontFamily = "monospace";
        elm.style.margin = "2px 0 2px 0";
        if (Object.prototype.toString.call(msg) === "[object Array]") {
            msg = msg.join();
        } else if (typeof msg === "object") {
            msg = msg === null ? "null" : JSON.stringify(msg);
        }
        elm.appendChild(document.createTextNode(msg));
        document.body.appendChild(elm);
    },

    // Writes out the given HTML at the end of the body,
    // exactly as-is
    logHTML: function(html) {
        document.body.insertAdjacentHTML("beforeend", html);
    }
};
</code></pre> 
             <p>I've made a version available <a href="http://tjcrowder.github.io/simple-snippets-console/snippet.js" rel="nofollow">here on GitHub Pages</a> for use until snippets have their own way of doing it. Feel free to insert that at the top of a snippet like this:</p> 
             <pre><code>&lt;!-- Provides the `snippet` object, see http://meta.stackexchange.com/a/242144/134069 --&gt;
&lt;script src="http://tjcrowder.github.io/simple-snippets-console/snippet.js"&gt;&lt;/script&gt;
</code></pre> 
             <p>A simple <code>snippet</code> object like that solves the issues with what people are doing now. Consider this example:</p> 
             <h3>Question:</h3> 
             <p>How do I create an array with just a subset of the contents of an existing array in JavaScript? For instance, only non-negative numbers?</p> 
             <h3>Answer:</h3> 
             <p>You can use ES5's <code>Array#filter</code> for that. It calls a function you supply, passing in each entry in the array, and builds a new array from the entries you return a truthy value for:</p> 
             <p></p>
             <div class="snippet" data-lang="js" data-hide="false"> 
              <div class="snippet-code"> 
               <pre class="snippet-code-js lang-js prettyprint-override"><code>var a = [10, 0, -5, 7, 12, -3, -1];
var filtered = a.filter(function(entry) {
  return entry &gt;= 0;
});
snippet.log("original: " + a.join(", "));        // "10, 0, -5, 7, 12, -3, -1"
snippet.log("filtered: " + filtered.join(", ")); // "10, 0, 7, 12"</code></pre> 
              </div> 
             </div> 
             <p></p> 
             <p><em>(Obviously if you try to run that, you'll get an error, as <code>snippet</code> isn't defined.)</em></p> 
             <p>Isn't that better than:</p> 
             <p></p>
             <div class="snippet" data-lang="js" data-hide="false"> 
              <div class="snippet-code"> 
               <pre class="snippet-code-js lang-js prettyprint-override"><code>var a = [10, 0, -5, 7, 12, -3, -1];
var filtered = a.filter(function(entry) {
  return entry &gt;= 0;
});
display("original: " + a.join(", "));        // "10, 0, -5, 7, 12, -3, -1"
display("filtered: " + filtered.join(", ")); // "10, 0, 7, 12"

function display(msg) {
    document.body.insertAdjacentHTML(
        "beforeend",
        "&lt;p&gt;" + msg + "&lt;/p&gt;"
    );
}</code></pre> 
              </div> 
             </div> 
             <p></p>

            <div class="post-text" itemprop="text"> 
             <p>After a long time, I've decided to post my answer.</p> 
             <p>I've started it in April this year and I've improved it these days.</p> 
             <p>My code, currently, takes control over the <code>console</code> object in a 'different' way (I'm missing the right word).</p> 
             <p>The code handles the following methods in the <code>console</code> object:</p> 
             <ul> 
              <li><code>log</code></li> 
              <li><code>error</code></li> 
              <li><code>warn</code></li> 
              <li><code>info</code></li> 
              <li><code>time</code></li> 
              <li><code>timeEnd</code></li> 
              <li><code>clear</code></li> 
             </ul> 
             <p>Also, it shows objects and arrays in a tree-like view.</p> 
             <p>A few examples of what the code can do:</p> 
             <p></p>
             <div class="snippet" data-lang="js" data-hide="true"> 
              <div class="snippet-code snippet-currently-hidden"> 
               <pre class="snippet-code-js lang-js prettyprint-override"><code>//starts the console and tells to use the specific element to add the messages there

console._RELAY_TO_DOC = document.getElementById('messages');

//misc stuff, only _USE_JSON has the default value

console._USE_JSON = false;
console._SHOW_COOL_NUMBERS = true;
console._SHOW_TYPES = true;
console._SHOW_STRING_LENGTH = true;

//just use the console now:

console.time('test');

console.log(1,[],{},'string',[1,2,3,4,5],{a:1,b:2,c:3});
console.warn('Too much awesomeness!');
console.info('You are informed.');
console.error('Oh no! We divided by 0!');

console.timeEnd('test');

throw new Error('This will show in the console');</code></pre> 
               <pre class="snippet-code-css lang-css prettyprint-override"><code>* {font-family:sans-serif;}

html,document {overflow:hidden;}

  max-height:100%;
}

#show:checked+#console{right:0px;}

  display:inline-block;

textarea {width:350px; height:130px;}</code></pre> 
               <pre class="snippet-code-html lang-html prettyprint-override"><code>&lt;!-- load the script --&gt;

&lt;/div&gt;

&lt;div&gt;
  Eval you code here: (write and click outside)&lt;br&gt;
  
  &lt;textarea onchange="Function(this.value)();"&gt;console.time('test');

console.warn('Too much awesomeness!');
console.info('You are informed.');

console.timeEnd('test');

throw new Error('This will show in the console');&lt;/textarea&gt;
&lt;/div&gt;</code></pre> 
              </div> 
             </div> 
             <p></p> 
             <p>I've asked <a href="http://meta.stackexchange.com/users/159427/canon">@canon</a> to lend me an important feature of his code: the ability to handle exceptions when uncaught.</p> 
             <p>There are also a few options in the <code>console</code> object:</p> 
             <ul> 
              <li><code>_RELAY_TO_DOC</code> - Enables/disables the logging of the messages in the document.<br> You can set it to <code>true</code> or set it with an <code>Element</code>.</li> 
              <li><code>_USE_JSON</code> - Simply returns a JSON string. Overrides all and every option from now on.</li> 
              <li><code>_SHOW_COOL_NUMBERS</code> - Show numbers in binary, octal and hexadecimal representation</li> 
              <li><code>_SHOW_TYPES</code> - Show the data type of an element.</li> 
              <li><code>_SHOW_STRING_LENGTH</code> - Shows the string length.</li> 
             </ul> 
             <p>It also adds 2 methods to the <code>console</code> object:</p> 
             <ul> 
              <li><code>_restore_old_method</code> - Restores a method (e.g.: <code>log</code>) to the default value</li> 
              <li><code>_restore_relay_method</code> - Does the oposite of the above method</li> 
             </ul> 
             <p>I hope you like it!</p> 
             <p>You can visit <a href="http://ismael-miguel.github.io/console-log-to-document/" rel="nofollow">http://ismael-miguel.github.io/console-log-to-document/</a> for a full explanation of the code and it's functionality.</p> 
             <p>To make it work in any page, just paste <code>&lt;script src="http://ismael-miguel.github.io/console-log-to-document/files/console.log.min.js"&gt;&lt;/script&gt;</code> into your <code>&lt;head&gt;</code>.</p> 
             <p><strong>Warning:</strong></p> 
             <ul> 
              <li>This <strong>does not</strong> like recursion (it is safe to use with <code>Element</code>s, it is prepared to ignore it's recursion)</li> 
              <li>The code can't handle very huge elements (like <code>document</code>)</li> 
              <li>This may burn a lot of CPU if misused</li> 
              <li>The <code>time</code> and <code>timeEnd</code> aren't accurate, but close enough to the <em>real</em> result</li> 
             </ul>
