            <div class="post-text" itemprop="text"> 
             <p>In this <a href="http://stackoverflow.com/questions/6268205/why-doesnt-ssis-recognize-line-feed-lf-row-delimiter-while-importing-utf-8-fl">SO answer</a>, I wanted to use the header <code>Script component code in C#</code> using the section title. However, the markdown syntax removed the hash sign (<code>#</code>) and it currently displays the text as <em><code>Script component code in C</code></em>. The only option that worked was ending the text with some character other than hash sign (<code>#</code>) like period (<code>.</code>).</p> 
             <h3>Things I have tried:</h3> 
             <p>Here are the list of different combinations I have tried based on the escape characters I found on this <a href="http://meta.stackexchange.com/a/29064/210877">MSO post</a>. I couldn't make any of these work as I expected it to.</p> 
             <pre><code>###Script component code in C\#
###Script component code in C`#
###Script component code in C*#
###Script component code in C_#
###Script component code in C{#}
###Script component code in C[#]
###Script component code in C(#)
###Script component code in C##
###Script component code in C+#
###Script component code in C-#
###Script component code in C.#
###Script component code in C!#
###Script component code in C ##
###Script component code in C#.
</code></pre> 
             <h3>Results:</h3> 
             <p><img src="http://i.stack.imgur.com/8jc6j.png" alt="Result screenshot"></p> 
             <h3>Searches on MSO:</h3> 
             <p><a href="http://meta.stackexchange.com/questions/33986/how-do-i-escape-in-code">http://meta.stackexchange.com/questions/33986/how-do-i-escape-in-code</a></p> 
             <p>This question seems to be about escaping hash sign in code and not the section title. I could not find other references when I searched for <code>markdown hash sign</code>. </p> 
             <h3>Question:</h3> 
             <p>How do I escape the hash sign if I want to use it in the section title?</p> 
             <h3>Reason why I am asking now:</h3> 
             <p>To answer <a href="http://meta.stackoverflow.com/users/191410/cyborgx37">@Cyborgx37</a>'s comment:</p> 
             <p>I rewrote the answer on February 8, 2013 completely with the latest release of the software to keep the answer up to date and also added more details to be more useful.</p>

            <div class="post-text" itemprop="text"> 
             <p>You can use the other format for producing headers, by "underlining" the heading in question: </p> 
             <pre><code>Script component code in C#
---------------------------- 
</code></pre> 
             <p>or </p> 
             <pre><code>Script component code in C#
=============================
</code></pre> 
             <p>this produces </p> 
             <blockquote> 
              <h2>Script component code in C#</h2> 
             </blockquote> 
             <p>Underlining with <code>-</code> produces a level 2 <code>h2</code> heading, while <code>=</code> produces a level 1 <code>h1</code> heading. Admittedly neither of these are the <code>h3</code> you're looking for, but they're all headings, you know. Semantically it's more accurate to use <code>h2</code> as your first level subheading, since your question title is a <code>h1</code>. </p> 
             <p>Or you could just use HTML: </p> 
             <pre><code>&lt;h3&gt;Script component code in C#&lt;/h3&gt;
</code></pre> 
             <p>which produces </p> 
             <blockquote> 
              <h3>Script component code in C#</h3> 
             </blockquote>

           <td class="answercell"> 
            <div class="post-text" itemprop="text"> 
             <p>The markdown parser allows for any number of closing hash characters right before the newline, which are ignored.</p> 
             <p>You could simply add a space after the hash but before the newline, which is practically indistinguishable from your desired output:</p> 
             <blockquote> 
              <h3>Script component code in C#</h3> 
             </blockquote> 
             <p>In case you want to try figuring out a different way to skin this cat, you can have a look at the regex being used in the PageDown source:</p> 
             <pre><code>/
    ^(\#{1,6})      // $1 = string of #'s
    [ \t]*
    (.+?)           // $2 = Header text
    [ \t]*
    \#*             // optional closing #'s (not counted)
    \n+
/gm
</code></pre>
